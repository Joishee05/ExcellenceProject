style.css - The Complete Styling System

What is style.css?

This 1,926-line file is where all the visual magic happens. Every color, every spacing decision, every animation - it's all defined here. Without this file, the app would just be black text on a white background.

Think of it like this

If you're building a house:

  • HTML - The walls, floors, and basic structure
  • CSS - The paint, flooring, light fixtures, and furniture
  • JavaScript - The electrical wiring and plumbing that makes things work

CSS Variables (Lines 2-60)

The file kicks off with CSS variables - these are like setting up a color palette before you start painting:

:root {
    --slu-navy: #003366;
    --slu-blue: #1e4a72;
    --slu-gold: #FFD700;
    --slu-white: #FFFFFF;
    
    --primary: var(--slu-navy);
    --secondary: var(--slu-gold);
    --background: #ffffff;
    --text-primary: #1e293b;
}

How CSS variables work

  • They let you store values and reuse them anywhere in your CSS
  • Always start with two dashes like --my-variable
  • Access them using var(--my-variable)
  • Defined in :root so they're available everywhere

The SLU color scheme

  • --slu-navy: #003366 - The main dark blue you see throughout
  • --slu-blue: #1e4a72 - A lighter blue for variety
  • --slu-gold: #FFD700 - The gold accent that makes things pop
  • These come straight from Saint Louis University's official brand colors

Why Use CSS Variables?

Without variables: If you want to change the primary color, you'd have to find and replace it in 50+ places.

With variables: Change --primary once, and it updates everywhere automatically!

Box Shadows (Lines 31-38)

--shadow-xs: 0 1px 2px 0 rgba(0, 51, 102, 0.05);
--shadow-sm: 0 1px 3px 0 rgba(0, 51, 102, 0.1);
--shadow-md: 0 4px 6px -1px rgba(0, 51, 102, 0.1);
--shadow-lg: 0 10px 15px -3px rgba(0, 51, 102, 0.1);
--shadow-xl: 0 20px 25px -5px rgba(0, 51, 102, 0.1);

These create depth effects with different intensity levels:

  • --shadow-xs - Very subtle, barely visible
  • --shadow-sm - Small, for buttons and inputs
  • --shadow-md - Medium, for cards
  • --shadow-lg - Large, for modals
  • --shadow-xl - Extra large, for important elements

Typography (Lines 49-51)

--font-primary: 'Inter', -apple-system, sans-serif;
--font-display: 'Crimson Text', Georgia, serif;
  • Inter: Modern sans-serif font for body text (clean, readable)
  • Crimson Text: Elegant serif font for headings (traditional, academic)
  • Fallbacks: If custom fonts fail to load, browser uses system fonts

Global Resets (Lines 54-85)

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

body {
    font-family: var(--font-primary);
    background-color: var(--background);
    color: var(--text-primary);
    line-height: 1.6;
    -webkit-font-smoothing: antialiased;
}

Why Reset Styles?

  • Different browsers have different default styles
  • Resetting creates a consistent starting point
  • box-sizing: border-box makes width calculations easier
  • -webkit-font-smoothing makes text look smoother on Mac/iOS

Header Styling (Lines 100-180)

The header uses Flexbox for layout:

#badge {
    display: flex;
    justify-content: space-between;
    align-items: center;
    background: var(--gradient-primary);
    padding: 1rem 2rem;
    box-shadow: var(--shadow-md);
}
  • display: flex - Enables flexbox layout
  • justify-content: space-between - Spreads items with space between
  • align-items: center - Vertically centers items
  • background: var(--gradient-primary) - Navy to blue gradient

Dropdown Styling (Lines 300-450)

The three cascading dropdowns (Boneset → Bone → Sub-bone) use custom styling:

select {
    width: 100%;
    padding: 0.75rem 1rem;
    border: 2px solid var(--border);
    border-radius: 8px;
    background-color: var(--background);
    font-size: 1rem;
    transition: all 0.3s ease;
}

select:hover {
    border-color: var(--primary);
}

select:focus {
    outline: none;
    border-color: var(--primary);
    box-shadow: 0 0 0 3px var(--primary-light);
}

Focus States for Accessibility

  • :hover - Changes appearance when mouse hovers
  • :focus - Changes when keyboard user tabs to element
  • Blue outline helps keyboard users see where they are
  • Never remove focus styles! Keyboard users need them

Content Grid Layout (Lines 600-750)

The side-by-side panels use CSS Grid:

.content-grid {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 2rem;
    margin-top: 2rem;
}

@media (max-width: 768px) {
    .content-grid {
        grid-template-columns: 1fr;
    }
}
  • grid-template-columns: 1fr 1fr - Two equal columns
  • 1fr = "1 fraction" (equal space)
  • gap: 2rem - Space between columns
  • Responsive: On mobile, switches to single column

Grid vs Flexbox

  • Flexbox: One-dimensional (row OR column)
  • Grid: Two-dimensional (rows AND columns)
  • Use Flexbox for: Navigation bars, button groups
  • Use Grid for: Page layouts, card galleries

Responsive Design (Lines 1700-1926)

The file ends with media queries for different screen sizes:

@media (max-width: 1024px) {
    .container {
        padding: 1.5rem;
    }
}

@media (max-width: 768px) {
    #badge {
        flex-direction: column;
        padding: 1rem;
    }
    
    .content-grid {
        grid-template-columns: 1fr;
    }
}

@media (max-width: 480px) {
    body {
        font-size: 14px;
    }
}

Breakpoints Explained

  • 1024px - Tablets in landscape
  • 768px - Tablets in portrait, large phones
  • 480px - Small phones
  • Styles "cascade down" - smaller screens inherit larger screen styles

Animations & Transitions

Smooth animations make the interface feel polished:

button {
    transition: all 0.3s ease;
}

button:hover {
    transform: translateY(-2px);
    box-shadow: var(--shadow-lg);
}

@keyframes fadeIn {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}
  • transition - Smoothly animates changes over time
  • transform: translateY(-2px) - Lifts element up 2 pixels
  • @keyframes - Defines custom animations
  • fadeIn animation makes content appear smoothly

Key CSS Concepts Used

  • CSS Variables: Reusable values for consistency
  • Flexbox: One-dimensional layouts (header, navigation)
  • Grid: Two-dimensional layouts (content panels)
  • Media Queries: Responsive design for all devices
  • Pseudo-classes: :hover, :focus, :active for interactivity
  • Transitions: Smooth property changes
  • Animations: Complex multi-step effects
  • Box Shadow: Depth and elevation
  • Border Radius: Rounded corners
  • Gradients: Color transitions

Professional Practices

  • BEM Naming: Block__Element--Modifier pattern for class names
  • Mobile-First: Base styles for mobile, media queries add desktop features
  • Accessibility: Focus states, sufficient color contrast
  • Performance: Hardware-accelerated transforms, efficient selectors
  • Maintainability: Variables make updates easy
  • Consistency: Reusable spacing, color, shadow values
