boneset.html - The Main HTML File

What is boneset.html?

This is the main HTML file that builds the entire user interface. It's essentially the skeleton of the app - everything users see and click on starts here. The file defines the structure and layout of the page.

Here's a simple way to think about it

Imagine you're building a restaurant:

  • boneset.html - The actual building structure: walls, doors, where the tables go
  • style.css - The paint, furniture, and decorations that make it look nice
  • JavaScript files - The staff who take orders and serve food
  • server.js - The kitchen where everything actually gets prepared

What's in this file

At 217 lines, this file sets up everything you see on screen:

  • Top navigation bar with all the main buttons
  • Search bar so users can quickly find specific bones
  • Three connected dropdown menus that filter from broad to specific
  • Description area where bone information appears
  • Image viewer to display bone photos
  • Navigation arrows to move between bones
  • Help popup that explains how everything works

Technologies we're using

  • HTML5: Modern semantic tags that tell browsers exactly what each section does
  • HTMX: Lets us add dynamic features right in the HTML without writing tons of JavaScript
  • ARIA attributes: Makes sure screen readers can understand and navigate the page
  • ES6 Modules: The current standard for organizing JavaScript code

Why we built it this way

This file follows industry best practices:

  • Semantic HTML: We use tags like <header> and <section> instead of just throwing <div> everywhere
  • Accessibility: People using screen readers or keyboards can still navigate everything
  • Progressive enhancement: The basic version works even without JavaScript enabled
  • Mobile-ready: The viewport settings make sure it looks good on phones and tablets

Let's walk through the file

1. Document Declaration (Lines 1-2)

<!DOCTYPE html>
<html lang="en">

What's happening here:

  • <!DOCTYPE html> - This tells the browser we're using HTML5
  • lang="en" - Specifies that the content is in English
  • These should always be the first lines in any HTML file

Why it matters: Without the DOCTYPE, browsers can go into "quirks mode" where they render things differently and unpredictably. The language attribute helps screen readers pronounce words correctly and lets search engines know what language you're writing in.

Breaking down each part:

<meta charset="UTF-8" />

  • Sets character encoding to UTF-8 (supports all languages and symbols)
  • Without this, special characters like é, ñ, or emoji might break
  • UTF-8 is the universal standard for modern websites

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

  • Most important line for mobile devices!
  • width=device-width - Makes the page width match the screen width
  • initial-scale=1.0 - Starts at 100% zoom (no zooming in/out by default)
  • Without this, mobile phones would show a tiny zoomed-out desktop version

<title>

  • Sets the text in the browser tab
  • Used as default bookmark name
  • Important for SEO (search engine optimization)

Google Fonts Loading

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  • preconnect - Starts connecting to Google's servers early (faster loading)
  • Loads "Inter" (modern sans-serif) and "Crimson Text" (elegant serif) fonts
  • Different font weights: 300 (light), 400 (normal), 500 (medium), 600 (semi-bold), 700 (bold)

External Resources

  • <link rel="stylesheet" href="style.css" /> - Links the CSS file for styling
  • <script src="https://unpkg.com/htmx.org@1.9.10"> - Loads HTMX library from CDN
  • HTMX version 1.9.10 is specifically chosen for stability

What is HTMX?

HTMX is a modern library that lets you add AJAX, transitions, and dynamic updates using just HTML attributes - no JavaScript required!

Example: Instead of writing 20 lines of JavaScript to fetch data, you write:

<button hx-get="/api/data" hx-target="#result">Load Data</button>

HTMX handles the fetch, response, and inserting content automatically!

Key concepts explained:

Semantic HTML

  • <header> - Semantic element meaning "this is the page header"
  • Better than <div class="header"> because it has meaning
  • Screen readers announce "banner landmark" when entering this section

ARIA Attributes (Accessibility)

  • role="banner" - Explicitly states this is the main header
  • role="navigation" - Marks the nav links section
  • role="button" - Tells screen readers these spans act as buttons
  • tabindex="0" - Makes elements keyboard-navigable (press Tab to focus)
  • aria-label="Toggle navigation menu" - Describes button purpose for screen readers

Why Use <span> Instead of <button>?

These are styled as text buttons for a cleaner look. The role="button" and tabindex="0" attributes make them functionally equivalent to real buttons for accessibility.

Hamburger Menu Icon (☰)

The "☰" symbol is a Unicode character (U+2630) called "trigram for heaven". It's commonly used as a menu icon on mobile sites.

On mobile devices, this button would toggle the sidebar navigation open/closed.

4. Search Section (Lines 54-71)

<section class="search-section">
    <div class="form-group">
        <label for="search-bar">Search Bones and Sub-bones</label>
        <div class="search-container">
            <input type="text" 
                   id="search-bar" 
                   placeholder="Enter bone name (minimum 2 characters)"
                   autocomplete="off" />
            <div id="search-loading" class="search-loading" style="display: none;">
                Searching...
            </div>
        </div>
    </div>
    
    <ul id="search-results" class="search-results" role="listbox" 
        aria-label="Search results"></ul>
</section>

Understanding each element:

<label for="search-bar">

  • Connects the label text to the input field
  • The for attribute MUST match the input's id
  • Clicking the label focuses the input (better UX)
  • Screen readers read the label when the input is focused
  • Always use labels with inputs! Never skip them!

<input type="text">

  • type="text" - Creates a single-line text input
  • placeholder - Shows hint text (disappears when typing)
  • autocomplete="off" - Prevents browser from showing previous search history
  • id - Must be unique on the entire page

Loading Indicator

  • style="display: none;" - Hidden by default
  • JavaScript will show/hide this during searches
  • Provides visual feedback that search is happening

Search Results

  • <ul> - Unordered list (results will be added here dynamically)
  • role="listbox" - ARIA role for selectable list
  • aria-label - Describes the list for screen readers

5. Dropdown Menus (Lines 74-114)

<section class="selection-section">
    <div class="form-group">
        <label>Select Anatomical Structure</label>
        <div class="form-row">
            <div class="select-group">
                <label for="boneset-select">Boneset</label>
                <select id="boneset-select" 
                        name="boneId"
                        hx-get="http://127.0.0.1:8000/api/description/" 
                        hx-target="#description-Container"
                        hx-trigger="change">
                    <option value="">Select a boneset</option>
                </select>
            </div>
            
            <div class="select-group">
                <label for="bone-select">Bone</label>
                <select id="bone-select" 
                        name="boneId"
                        hx-get="http://127.0.0.1:8000/api/description/" 
                        hx-target="#description-Container"
                        hx-trigger="change"
                        disabled>
                    <option value="">Select a bone</option>
                </select>
            </div>
            
            <div class="select-group">
                <label for="subbone-select">Sub-bone</label>
                <select id="subbone-select"
                        name="boneId"
                        hx-get="http://127.0.0.1:8000/api/description/" 
                        hx-target="#description-Container"
                        hx-trigger="change"
                        disabled>
                    <option value="">Select a sub-bone</option>
                </select>
            </div>
        </div>
    </div>
</section>

This is where HTMX magic happens!

Dependent Dropdowns

  • Boneset dropdown: Always enabled, populated on page load
  • Bone dropdown: Starts disabled, enables when boneset is selected
  • Sub-bone dropdown: Starts disabled, enables when bone is selected
  • This creates a cascading selection flow

HTMX Attributes Explained

  • hx-get="http://127.0.0.1:8000/api/description/"
    • Makes a GET request to this URL
    • Automatically includes the selected value
    • 127.0.0.1:8000 is the local development server
  • hx-target="#description-Container"
    • Puts the server response into the element with id="description-Container"
    • Replaces existing content automatically
  • hx-trigger="change"
    • Triggers the request when selection changes
    • No "Submit" button needed!

How Dependent Dropdowns Work

Step 1: User selects "Skull" from Boneset dropdown

Step 2: JavaScript file (dropdowns.js) detects the change

Step 3: It fetches bones that belong to "Skull" from the server

Step 4: It enables the Bone dropdown and fills it with options

Step 5: User selects a bone, process repeats for Sub-bone dropdown

This creates a smooth, guided user experience!

Side-by-side Layout:

  • content-grid - CSS class that creates a two-column layout
  • Left panel: Text descriptions
  • Right panel: Images and visual references
  • Responsive: Stacks vertically on mobile devices

ARIA Live Regions

  • aria-live="polite" - Announces content changes to screen readers
  • "polite" means it waits for user to finish current action before announcing
  • Alternative: aria-live="assertive" interrupts immediately (use for urgent alerts)

Placeholder Content

  • Shows instructional text before user selects anything
  • Hidden by CSS when real content loads
  • Provides clear guidance on what to do

9. JavaScript Module Loading (Lines 208-212)

<script type="module" src="js/sidebar.js"></script>
<script type="module" src="js/main.js"></script>
<script type="module" src="js/imageDisplay.js"></script>
<script type="module" src="js/annotationOverlay.js"></script>
  • type="module" - ES6 modules (modern JavaScript)
  • Each file handles one specific responsibility
  • Modules load defer automatically (after HTML parses)
  • Can use import/export syntax

Why Separate JavaScript Files?

Separation of Concerns: Each file does one job:

  • sidebar.js: Handles mobile menu toggle
  • main.js: Coordinates everything, initializes app
  • imageDisplay.js: Loads and displays bone images
  • annotationOverlay.js: Handles image annotations/labels

This makes code easier to debug, test, and maintain!
