HTMX Targets & Swapping

Lesson 21 of 23

What is a Target?

A target is the HTML element where you want to place the response from the server. By default, HTMX replaces the element that made the request, but you can change this with the hx-target attribute.

Using hx-target

The hx-target attribute lets you specify where the response should go using CSS selectors:

<button hx-get="/load-content" hx-target="#result"> Load Content </button> <div id="result"></div>

When you click the button, the server response replaces the content inside the div with id="result", not the button itself.

Target Selectors

You can target elements using any CSS selector:

  • hx-target="#myId" - Target element with specific id
  • hx-target=".myClass" - Target element with specific class
  • hx-target="div" - Target the first div element
  • hx-target="this" - Target the element itself (default behavior)
  • hx-target="closest div" - Target the closest parent div

Understanding hx-swap

The hx-swap attribute controls HOW the response is inserted into the target element. Here are the common options:

Common Swap Strategies

Swap Strategy Description Use Case
innerHTML Replace the contents of the target element Update content inside a container
outerHTML Replace the entire target element Replace a div or section completely
beforebegin Insert before the target element Add element above current element
afterbegin Insert at the start of target's content Add to top of a list
beforeend Insert at the end of target's content Add to bottom of a list (most common)
afterend Insert after the target element Add element below current element

Real-World Examples

Example 1: Add Item to List

When adding a new todo, you want to append it to the bottom of the list:

<button hx-post="/add-todo" hx-target="#todos" hx-swap="beforeend"> Add Todo </button> <ul id="todos"> <li>Learn HTMX</li> <li>Build a project</li> </ul>
Result: New todos are added to the end of the list without replacing existing items.

Example 2: Replace Entire Section

Update a user profile section completely:

<button hx-get="/user-profile" hx-target="#profile" hx-swap="innerHTML"> Refresh Profile </button> <div id="profile"> <h2>User Profile</h2> <p>Name: John</p> </div>
Result: The entire content of the profile div gets replaced, but the div itself stays in place.

Example 3: Delete with Removal

Remove an item from a list after deletion:

<li> Learn CSS <button hx-delete="/todos/123" hx-target="this" hx-swap="outerHTML swap:1s">Delete</button> </li>
Result: Clicking delete removes the entire list item. The swap:1s adds a 1-second delay.

Combining Target and Swap

The real power comes from combining hx-target and hx-swap:

<form hx-post="/search" hx-target="#results" hx-swap="innerHTML"> <input type="text" name="query" placeholder="Search..."> <button>Search</button> </form> <div id="results"></div>

This form posts to /search and replaces the contents of #results with the server's response.

Advanced: Out of Band Swaps

Sometimes you want to update multiple elements with a single request. Use hx-swap-oob:

<!-- Button that updates multiple areas --> <button hx-post="/update">Update</button> <!-- Server returns multiple elements to update --> <div id="main" hx-swap-oob="true">Main content</div> <div id="sidebar" hx-swap-oob="true">Sidebar content</div>

Summary

  • hx-target specifies WHERE to put the response
  • hx-swap specifies HOW to insert it
  • Common swaps: innerHTML, outerHTML, beforeend, afterbegin
  • Use CSS selectors to target any element
  • Combine them for powerful, precise DOM updates
