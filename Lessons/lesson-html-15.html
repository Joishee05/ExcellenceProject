HTML Best Practices

Lesson 15 of 15 • Final Lesson

Congratulations on completing all 15 HTML lessons! This final lesson covers essential best practices that will help you write clean, professional, accessible, and maintainable HTML code. These habits will make you a better web developer.

Code Organization & Structure

Use Proper Indentation

Indentation makes code readable and easier to debug. Use consistent spacing (2 or 4 spaces):

<!-- Good indentation -->
<html>
  <head>
    <title>Page</title>
  </head>
  <body>
    <h1>Welcome</h1>
  </body>
</html>

Use Semantic HTML

Choose meaningful tags that describe content, not just visual appearance:

  • Use <header>, <nav>, <main>, <footer>
  • Use <h1>-<h6> for headings (don't skip levels)
  • Use <p> for paragraphs, not <div> with styling
  • Use <button> for buttons, not <div> with click handlers

Accessibility (A11y)

Always Include Alt Text for Images

<!-- Good -->
<img src="dog.jpg" alt="Golden Retriever playing in park">

<!-- Bad -->
<img src="dog.jpg" alt="dog">

Use Semantic Labels for Forms

<!-- Good -->
<label for="email">Email:</label>
<input type="email" id="email" name="email">

<!-- Bad -->
<input type="email" placeholder="email">

Provide Descriptive Link Text

<!-- Good -->
<a href="about.html">Read our About page</a>

<!-- Bad -->
<a href="about.html">Click here</a>

Performance Best Practices

Optimize Images

  • Use modern formats: WebP (with fallback to PNG/JPG)
  • Compress images before uploading
  • Use appropriate dimensions (don't load 4000px image for 400px display)
  • Consider responsive images with srcset

Minimize External Resources

  • Load CSS in <head>
  • Load JavaScript at end of <body> or use defer
  • Minimize HTTP requests
  • Use CSS instead of images when possible

Meta Viewport Tag

Always include for mobile responsiveness:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Security Best Practices

Validate Form Input

Use HTML5 input types and required attributes:

<input type="email" required>
<input type="password" minlength="8" required>
<input type="number" min="0" max="100">

Use HTTPS

Always serve content over HTTPS, especially forms handling sensitive data.

Don't Hardcode Sensitive Data

Never include passwords, API keys, or tokens in HTML.

Content & Naming Conventions

Use Clear, Descriptive Class Names

<!-- Good -->
<div class="product-card">
<div class="primary-button">

<!-- Bad -->
<div class="red-box">
<div class="btn1">

Use Lowercase for Tags & Attributes

<!-- Good -->
<div id="main-content" class="container">

<!-- Bad -->
<DIV ID="Main-Content" CLASS="Container">

Use IDs for Unique Elements, Classes for Reusable Styles

<div id="header"> <!-- Unique -->
<div class="button"> <!-- Reusable -->

Common HTML Mistakes to Avoid

✗ Common Mistakes
  • ✗ Missing alt text on images
  • ✗ Using outdated HTML tags
  • ✗ Not closing tags properly
  • ✗ Skipping h1 or h2 levels
  • ✗ Using tables for layout
  • ✗ Missing DOCTYPE declaration
✓ Best Practices
  • ✓ Always use descriptive alt text
  • ✓ Use modern HTML5 elements
  • ✓ Validate HTML code
  • ✓ Use heading hierarchy properly
  • ✓ Use semantic HTML
  • ✓ Start with valid HTML5 structure

HTML Validation

Always validate your HTML to find errors:

  • W3C Validator: https://validator.w3.org/
  • Browser DevTools: Right-click → Inspect Element
  • VS Code Extensions: Use HTML Hint or similar

Testing & Quality Assurance

  • Test on different browsers (Chrome, Firefox, Safari, Edge)
  • Test on mobile devices or browser emulators
  • Use browser DevTools to check for console errors
  • Test keyboard navigation and screen readers
  • Check page load performance

Keep Learning

Next steps after HTML: Learn CSS to style your pages, then JavaScript to add interactivity. Combine HTML, CSS, and JavaScript to build complete web applications. Always stay updated with web standards.

HTML5 Quick Reference

Semantic: <header>, <nav>, <main>, <article>, <section>, <footer>

Forms: <form>, <input>, <textarea>, <select>, <button>

Media: <img>, <audio>, <video>, <source>

Structure: <table>, <ul>, <ol>, <li>, <div>

Final Key Takeaways
  • HTML is the foundation of all web development
  • Semantic HTML improves accessibility and SEO
  • Always include alt text, proper structure, and validation
  • HTML5 includes modern tags for better organization
  • Performance and security matter from day one
  • Write HTML for humans AND machines (browsers, search engines)
  • Keep learning and stay updated with web standards
  • Practice by building real projects
