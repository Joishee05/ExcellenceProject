Modules & Imports

Lesson 13 of 15

What You'll Learn

In this comprehensive lesson, you will learn about JavaScript modules and imports - the system that professional developers use to organize and reuse code. Modules are essential for building scalable, maintainable applications. By the end of this lesson, you'll understand how to break code into modules and import them when needed.

What Are Modules?

Modules are a way to organize JavaScript code into separate files. They let you break your code into manageable pieces and reuse them across your application. Every modern JavaScript project uses modules because they make code cleaner, more organized, and easier to maintain.

Before modules, developers had to put all their code in a single file or multiple files that all ran in the global scope, leading to naming conflicts and messy code. Modules solve this problem by creating isolated code environments.

Why Use Modules?

  • Organization: Related code stays together in its own file
  • Reusability: Use the same code in multiple places without duplication
  • Maintainability: Easier to find and fix bugs when code is organized
  • Scalability: Projects can grow without becoming chaotic
  • Testing: Easy to test individual modules in isolation
  • Namespace Isolation: Avoids naming conflicts by keeping variables private
  • Performance: Only load the code you actually need

Module Basics: Export and Import

The two core features of JavaScript modules are export and import. They work together to let different files share code:

Key Concept: How Modules Work
  • export - Makes code available for other files to use
  • import - Brings code from other modules into your current file
  • Named exports - Export multiple specific items with names
  • Default export - Export one main thing per module

Named Exports and Imports

Named exports let you export multiple functions, variables, or classes from a module. Each export must have a name, and when you import, you specify which items you want.

Example: Creating a Math Module

// math.js - This file exports two functions export function add(a, b) { return a + b; } export function subtract(a, b) { return a - b; } export function multiply(a, b) { return a * b; }

What This Does: The export keyword makes these functions available for other files to import and use.

Example: Importing Named Exports

// main.js - Import only the functions we need import { add, subtract } from './math.js'; console.log(add(5, 3)); // 8 console.log(subtract(10, 4)); // 6 // Note: multiply is not imported, so this would cause an error: // console.log(multiply(3, 4)); // Error!

What This Does: We use curly braces { } to import only the specific functions we need from the math module.

Default Exports and Imports

Default exports let you export one main thing from a module. When you import a default export, you don't need curly braces and you can name it whatever you want.

Example: Default Export

// calculator.js - Export one main calculator function export default function calculate(a, b, operation) { if (operation === 'add') return a + b; if (operation === 'subtract') return a - b; if (operation === 'multiply') return a * b; if (operation === 'divide') return a / b; return 'Unknown operation'; }

Example: Importing Default Export

// main.js - Import the default export import calculate from './calculator.js'; // Notice: no curly braces, and we can name it whatever we want console.log(calculate(10, 5, 'multiply')); // 50 console.log(calculate(20, 4, 'divide')); // 5

Key Differences: Default exports don't use curly braces and can be imported with any name you choose. You can only have one default export per module.

Exporting and Importing Objects

You can also export objects that contain related data and functions:

Example: Exporting an Object

// user.js - Export a user management module export const userManager = { users: [], addUser(name, email) { this.users.push({ name, email, id: Date.now() }); }, getUser(id) { return this.users.find(user => user.id === id); }, removeUser(id) { this.users = this.users.filter(user => user.id !== id); } };

Example: Importing and Using the Object

// app.js - Use the user manager import { userManager } from './user.js'; userManager.addUser('John', 'john@example.com'); userManager.addUser('Jane', 'jane@example.com'); console.log(userManager.users); // Shows all users const john = userManager.getUser(/* id */); console.log(john); // Shows John's data

Module Organization Strategies

Here are best practices for organizing your modules:

1. One Responsibility Per Module

Each module should do one thing well. Don't mix different functionality in a single file.

// Good: math.js contains only math functions export function add(a, b) { return a + b; } export function subtract(a, b) { return a - b; } // Good: string.js contains only string functions export function capitalize(str) { return str.charAt(0).toUpperCase() + str.slice(1); }

2. Clear Module Names

File names should clearly describe what the module contains:

  • userManager.js - Manages user data and operations
  • apiClient.js - Handles API requests
  • dateFormatter.js - Formats and parses dates
  • validators.js - Validates data and forms

3. Organize with Folders

For larger projects, group related modules into folders:

project/ ├── utils/ │ ├── math.js │ ├── string.js │ └── date.js ├── api/ │ ├── users.js │ └── posts.js ├── components/ │ ├── header.js │ └── button.js └── index.js (main entry point)

Combining Multiple Imports

You can import multiple named exports and a default export in one statement:

// Import named exports AND a default export import Calculator, { add, subtract } from './math-tools.js'; // Use them together console.log(add(5, 3)); // 8 console.log(Calculator.calculate()); // (default export)

Real-World Module Example

Here's how modules work in a real application:

// modules/storage.js - Handles local storage export function saveUser(user) { localStorage.setItem('user', JSON.stringify(user)); } export function getUser() { const user = localStorage.getItem('user'); return user ? JSON.parse(user) : null; } export function clearUser() { localStorage.removeItem('user'); }
// app.js - Uses the storage module import { saveUser, getUser, clearUser } from './modules/storage.js'; const user = { name: 'Alice', email: 'alice@example.com' }; saveUser(user); console.log(getUser()); // { name: 'Alice', email: 'alice@example.com' } clearUser(); console.log(getUser()); // null

Module Best Practices Summary

  • One Responsibility Per Module - Each file should do one thing and do it well
  • Descriptive Names - Use clear file names that describe the module's purpose
  • Organized Structure - Group related modules into folders for large projects
  • Prefer Named Exports - More explicit and easier to refactor than default exports
  • Document Your Modules - Add comments explaining what each export does
  • Keep Modules Focused - Don't try to do too much in one file
  • Test Individual Modules - Test each module separately before combining
