Lesson 14: Best Practices

Lesson 14 of 15

What You'll Learn

In this lesson, you will learn CSS best practices for writing clean, maintainable, performant stylesheets. These practices help you and your team build better websites faster.

Organize Your CSS

Logical Structure

Organize your CSS file logically.

/* 1. CSS Variables */ :root { --primary-color: #10b981; } /* 2. Reset/Normalize */ * { margin: 0; padding: 0; box-sizing: border-box; } /* 3. Base/Global Styles */ body { font-family: sans-serif; line-height: 1.6; } /* 4. Layout/Grid */ .container { max-width: 1200px; margin: 0 auto; } /* 5. Components */ .button { /* Button styles */ } /* 6. Utilities */ .text-center { text-align: center; }

Separate Concerns

Keep CSS organized by putting related styles together.

/* DON'T: Mix unrelated styles */ .container { width: 1200px; } .button { padding: 10px; } .container { margin: 0 auto; } /* Scattered */ /* DO: Group related styles */ .container { width: 1200px; margin: 0 auto; } .button { padding: 10px; background: #10b981; }

Naming Conventions

BEM (Block-Element-Modifier)

A system for naming CSS classes that makes them clear and organized.

/* Block - the main component */ .card { } /* Element - parts of the card, double underscore __ */ .card__header { } /* The header PART of the card */ .card__body { } /* The body PART of the card */ .card__footer { } /* The footer PART of the card */ /* Modifier - variations, double dash -- */ .card--highlighted { } /* A HIGHLIGHTED version of card */ .card--error { } /* An ERROR version of card */

Why This Works So Well:

  • .card: This is the main component (Block). The parent container
  • .card__header: Double underscore means "this is PART OF card" (Element). It's specifically the header part
  • .card--highlighted: Double dash means "this is a VARIATION of card" (Modifier). A different type or state of card
  • By reading the class name, you immediately know: what component it belongs to, what part it is, and what variation
  • No confusing cascade - every element has its own clear, descriptive name

Benefits of BEM

  • Clarity: Name clearly indicates element purpose
  • No Nesting: Avoids specificity issues
  • Scalability: Easy to add new variants

Alternative: Utility-First

/* Utility classes that do one thing */ .flex { display: flex; } .gap-md { gap: 16px; } .text-center { text-align: center; }

CSS Specificity

Specificity Rules

Selector Specificity
Element: p 0-0-1
Class: .button 0-1-0
ID: #header 1-0-0
Inline: style="" Very high
!important Highest

Best Practice

Keep specificity low by favoring classes over IDs and avoiding nesting.

/* DON'T: High specificity */ #header nav ul li a:hover { } /* DO: Low specificity */ .nav__link:hover { }

Avoid Common Mistakes

Overusing !important

/* DON'T */ .button { color: blue !important; } /* DO: Better specificity management */ .button { color: blue; } .button--primary { color: green; }

Deep Nesting

/* DON'T: Too deep, hard to maintain */ .container > div > div > div > p { } /* DO: Simpler, more specific class */ .section-title { }

Inline Styles

Performance Tips

Minimize CSS

Remove unnecessary selectors and properties.

/* DON'T: Unused styles */ .old-component { } div.container { } /* DO: Clean CSS */ .container { }

Use CSS Efficiently

  • Combine selectors: .button, .link instead of separate rules
  • Use shorthand: margin: 10px instead of margin-top, margin-bottom, etc.
  • Group media queries: Keep responsive styles together or at end

Avoid Expensive Properties

/* Performance: Good */ transform: translateX(10px); opacity: 0.5; /* Performance: Poor - causes reflows */ left: 10px; width: 90%;

Accessibility Considerations

Color Contrast

Ensure sufficient contrast between text and background.

/* Poor contrast */ color: #888; background: #f0f0f0; /* Fails WCAG AA */ /* Good contrast */ color: #333; background: #f0f0f0; /* Passes WCAG AA */

Focus States

Always provide visible focus indicators for keyboard navigation.

button:focus { outline: 3px solid #10b981; outline-offset: 2px; }

Respecting Preferences

@media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }

Documentation and Comments

Comment Complex Code

/* Main layout grid with specific breakpoints Adjusted for mobile-first approach See design file: layout-specs.fig */ .grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); }

Section Comments

/* ==================================== HEADER STYLES ==================================== */ /* ==================================== NAVIGATION ==================================== */

CSS Frameworks and Preprocessors

When to Use

  • Frameworks (Bootstrap, Tailwind): Quick prototyping, consistent components
  • Preprocessors (SASS, LESS): Variables, nesting, mixins for complex projects
  • Pure CSS: Custom projects, CSS variables, smaller codebases

Version Control for CSS

  • Track changes: Use git to track CSS modifications
  • Review PRs: Have CSS changes reviewed by team members
  • Document changes: Describe why CSS changed, not just what changed
