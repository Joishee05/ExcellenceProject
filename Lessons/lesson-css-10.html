Lesson 10: Responsive Design

Lesson 10 of 15

What You'll Learn

In this lesson, you will master responsive designcreating layouts that adapt beautifully to any screen size. You will learn mobile-first approach, flexible units, and media queries.

What is Responsive Design?

Responsive design ensures your website works well on all devices: phones, tablets, and desktops. Instead of building separate websites for different sizes, one responsive website adapts to its container.

Mobile-First Approach

Design for mobile first, then enhance for larger screens. This ensures core functionality works everywhere.

/* Mobile styles (base) */ .container { width: 100%; padding: 10px; } .column { width: 100%; /* Full width on mobile */ margin-bottom: 20px; } /* Tablet and up */ @media (min-width: 768px) { .column { width: 50%; margin-bottom: 0; } } /* Desktop and up */ @media (min-width: 1024px) { .column { width: 33.33%; } }

Flexible Units

Use relative units instead of fixed pixels for flexible layouts.

Unit What It Means Use Case
% Percentage of parent Container widths, flexible columns
em Relative to parent font size Font sizes, line-height
rem Relative to root font size Consistent sizing throughout
vw/vh 1% of viewport width/height Full-screen sections, responsive font

Using Flexible Units

body { font-size: 16px; /* Base font size (16px = 1rem) */ } h1 { font-size: 2rem; /* 32px on base size */ } .container { max-width: 1200px; /* Max width, but flexible below */ margin: 0 auto; padding: 0 5%; /* 5% padding on sides */ } .column { width: 50%; /* Half of parent */ padding: 1rem; /* Flexible padding */ }

Viewport Meta Tag

Always include the viewport meta tag in your HTML head for proper mobile rendering.

What This Does

  • width=device-width: Sets viewport width to device width (prevents zooming out on mobile)
  • initial-scale=1.0: Sets initial zoom level to 100%

Responsive Typography

Text should be readable on all screen sizes. Adjust font sizes for different devices.

/* Mobile - default */ h1 { font-size: 24px; line-height: 1.3; } p { font-size: 16px; line-height: 1.6; } /* Tablet */ @media (min-width: 768px) { h1 { font-size: 32px; } p { font-size: 18px; } } /* Desktop */ @media (min-width: 1024px) { h1 { font-size: 40px; } p { font-size: 20px; } }

Responsive Images

Images should scale with their containers to prevent overflow.

img { max-width: 100%; /* Never wider than container */ height: auto; /* Maintain aspect ratio */ display: block; /* Remove bottom spacing */ }

Common Breakpoints

Device Breakpoint
Mobile 0px - 600px
Tablet 601px - 1024px
Desktop 1025px+

Responsive Layout Examples

Responsive Two-Column Layout

/* Mobile: Stack vertically */ .main { width: 100%; margin-bottom: 20px; } .sidebar { width: 100%; } /* Tablet and up: Side by side */ @media (min-width: 768px) { .main { width: 70%; float: left; } .sidebar { width: 30%; float: right; } }

Responsive Navigation Menu

/* Mobile: Hidden by default */ .nav-menu { display: none; } .hamburger { display: block; } /* Tablet and up: Show menu */ @media (min-width: 768px) { .nav-menu { display: flex; gap: 30px; } .hamburger { display: none; } }

Testing Responsive Design

  • Browser DevTools: Use device emulation (F12 or right-click > Inspect)
  • Real Devices: Test on actual phones and tablets
  • Responsive Design Checklist: Touch targets 44x44px minimum, readable text at arm's length, fast load times
