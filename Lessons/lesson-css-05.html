Lesson 5: Layout & Display

Lesson 5 of 15

What You'll Learn

In this important lesson, you will master the display property and basic layout concepts. Understanding how elements are displayed is crucial for building any layout.

The Display Property

The display property controls how an HTML element is rendered. It affects how the element flows with other content and how it responds to width/height properties.

Display: Block

Block-level elements take up the full available width and start on a new line. Each block element occupies its own horizontal space.

display: block; /* Default block elements include: */ div, p, h1-h6, header, footer, section, article, ul, ol, blockquote

What Block Display Does:

  • display: block; - Makes the element take full width (stretches left to right)
  • New line: Starts on a new line and forces content after it to the next line
  • Stacking: Multiple block elements stack vertically like paragraphs in a document
  • Default block elements: p, div, h1-h6 are naturally block. That's why paragraphs don't sit side-by-side
  • Width and height work: You can set width: 200px on a block element and it actually becomes 200px

Characteristics of Block Elements

  • Takes up full available width (stretches to fill its container)
  • Starts on a new line and forces content after it to a new line
  • Respects all margin, padding, and border properties
  • Width and height properties work as expected

Block Element Example

.box { display: block; width: 100%; margin: 10px 0; padding: 20px; background: #f5f5f5; }

Display: Inline

Inline elements do not start on a new line and only take up as much width as necessary. They flow within text and other inline content.

display: inline; /* Default inline elements include: */ a, span, strong, em, img, button (mostly)

What Inline Display Does:

  • display: inline; - Makes element sit next to other content (no new line)
  • Only needed width: Takes only as much space as its content needs. If content is 50px wide, element is 50px
  • Flows like text: Multiple inline elements sit side-by-side like words in a sentence
  • Default inline elements: <a>, <span>, <strong>, <em> are naturally inline
  • Problem: Top/bottom margin doesn't work! Left/right margin/padding works but top/bottom is ignored
  • Width/height ignored: Can't set width: 200px on inline elements - it doesn't work

Characteristics of Inline Elements

  • Only takes up as much width as necessary
  • Does not start on a new line; flows within text
  • Top and bottom margin properties are ignored
  • Left and right margin/padding work, but top and bottom do not
  • Width and height properties do not apply
  • Line breaks may appear between inline elements

Inline Element Example

span { display: inline; background: yellow; margin-left: 10px; /* Works */ margin-top: 10px; /* Does NOT work on inline elements */ }

Display: Inline-Block

Inline-block is a hybridelements flow inline (no new line) but respect margin, padding, width, and height properties like block elements.

display: inline-block;

What Inline-Block Does:

  • display: inline-block; - The "best of both worlds" between block and inline
  • Flows inline: Multiple inline-block elements sit side-by-side like inline. No new line
  • Respects sizing: But width: 200px WORKS. You can set actual dimensions
  • Respects spacing: Top/bottom margin and padding work perfectly (unlike inline)
  • Perfect for buttons: Buttons next to each other with proper spacing and clickable areas
  • Real example: Navigation buttons, image galleries where items sit side-by-side

Characteristics of Inline-Block Elements

  • Flows inline with other content (no new line)
  • Respects width, height, margin, and padding (all sides)
  • Useful for creating buttons, badges, and small components
  • Can have whitespace issues due to HTML formatting

Inline-Block Example

.button { display: inline-block; padding: 10px 20px; margin: 5px; /* All sides work */ background: #0066cc; color: white; border-radius: 4px; }

Other Important Display Values

Display: None

Completely removes an element from the page flow as if it doesn't exist.

display: none; /* Element is completely hidden */ /* Different from visibility: hidden which hides element but keeps its space */

Display: Flex

Enables Flexbox layout. The container becomes a flex container, and children become flex items.

display: flex; /* Simple horizontal layout */ display: flex; flex-direction: column; /* Vertical layout */

Display: Grid

Enables CSS Grid layout for two-dimensional layouts.

display: grid; /* Two-dimensional layout */ grid-template-columns: repeat(3, 1fr); /* Three equal columns */

Practical Layout Examples

Navigation Bar with Inline-Block

nav { background: #333; padding: 10px; } nav a { display: inline-block; color: white; padding: 10px 15px; text-decoration: none; margin: 0 5px; } nav a:hover { background: #555; }

Creating a Simple Grid with Inline-Block

.item { display: inline-block; width: calc(33.333% - 10px); /* 3 items per row */ margin: 5px; padding: 15px; background: white; border: 1px solid #ddd; }

Best Practices for Display Property

  • Know Your Elements: Understand which elements are block, inline, or inline-block by default
  • Use Semantic Elements: Use appropriate HTML elements for their intended purpose
  • Prefer Flexbox for Modern Layouts: Flexbox is more powerful and flexible than inline-block
  • Use Grid for Complex Layouts: Grid excels at two-dimensional layouts
  • Be Explicit: Set display properties explicitly for clarity, even if using defaults

Responsive Display Changes

You can change the display property at different screen sizes for responsive design:

/* Desktop: horizontal navigation */ nav { display: flex; } /* Mobile: vertical navigation */ @media (max-width: 768px) { nav { display: flex; flex-direction: column; } }
