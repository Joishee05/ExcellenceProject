Lesson 8: Positioning

Lesson 8 of 15

What You'll Learn

In this lesson, you will master CSS positioninghow to place elements precisely on your page. You will learn static, relative, absolute, fixed, and sticky positioning, and how to use z-index to control layering.

The Position Property

The position property controls how an element is positioned in the document flow. There are five main values, each with different behavior.

position: static; /* Default - normal document flow */ position: relative; /* Offset from where it normally would be */ position: absolute; /* Removed from flow, positioned relative to parent */ position: fixed; /* Removed from flow, stuck to viewport (stays when scrolling) */ position: sticky; /* Starts normal, becomes fixed when scrolling */

What Each Position Does:

  • static (default): Elements appear in normal order. It's like reading a book - one line after another
  • relative: Element stays in normal flow but you can nudge it with top/left/right/bottom. Leaves a gap where it originally was
  • absolute: Element is removed from normal flow and positioned relative to its parent (if parent has position set). Like dragging an element to an exact spot
  • fixed: Element stays in exact position on screen even when scrolling. Great for sticky headers or floating buttons
  • sticky: Element scrolls normally but "sticks" to top when you scroll past it. Table headers often use this

Position: Static (Default)

Static is the default position value. Elements flow normally in the document.

position: static; /* Offset properties (top, left, etc.) are ignored */

Position: Relative

Relative positioning moves an element from its normal position while keeping its space in the document flow.

position: relative; top: 20px; /* Move 20px down from normal position */ left: 30px; /* Move 30px right from normal position */ right: 10px; /* Move 10px left (alternative to left) */ bottom: 15px; /* Move 15px up (alternative to top) */

Relative Positioning Example

.box { position: relative; top: 10px; left: 15px; /* Element moves 10px down and 15px right, but space remains */ }

Position: Absolute

Absolute positioning removes an element from the document flow and positions it relative to its nearest positioned ancestor (or the document if none exists).

position: absolute; top: 50px; left: 100px;

Creating a Positioned Container

To use absolute positioning effectively, create a positioned container:

.container { position: relative; /* This becomes the positioning context */ } .badge { position: absolute; top: 0; right: 0; /* Badge in top-right corner */ }

Absolute Positioning Example: Card Badge

.card { position: relative; padding: 20px; background: white; border: 1px solid #ddd; } .badge { position: absolute; top: -10px; right: -10px; width: 30px; height: 30px; background: red; color: white; border-radius: 50%; display: flex; align-items: center; justify-content: center; }

Position: Fixed

Fixed positioning positions an element relative to the viewport. It stays in place when scrolling.

position: fixed; top: 0; left: 0;

Common Fixed Positioning Uses

  • Sticky Headers: Navigation bars that stay at top while scrolling
  • Floating Buttons: Chat widgets, notification popups
  • Sticky Sidebars: Table of contents, ads

Fixed Header Example

header { position: fixed; top: 0; left: 0; right: 0; background: #333; z-index: 100; /* Keep above other content */ } body { padding-top: 80px; /* Space for fixed header */ }

Position: Sticky

Sticky is a hybrid of relative and fixed. An element starts in the normal flow but becomes fixed when scrolled to a specified offset.

position: sticky; top: 0; /* Stick when 0px from top */

Sticky Positioning Use Cases

  • Table Headers: Headers stick while scrolling table
  • Section Titles: Headers stick within sections
  • Floating Navigation: Secondary nav sticks on scroll

Sticky Table Header

thead { position: sticky; top: 0; background: #f5f5f5; border-bottom: 2px solid #333; }

Z-Index: Stacking Order

When elements overlap, z-index controls which appears on top. Higher z-index values appear in front.

z-index: 1; /* Lower - appears behind */ z-index: 100; /* Higher - appears in front */ z-index: 999; /* Very front */ /* z-index only works on positioned elements */

Z-Index Example: Modal Dialog

.modal { position: fixed; top: 50%; left: 50%; transform: translate(-50%, -50%); z-index: 1000; /* Above everything */ background: white; box-shadow: 0 4px 20px rgba(0,0,0,0.3); } .modal-backdrop { position: fixed; top: 0; left: 0; right: 0; bottom: 0; background: rgba(0,0,0,0.5); z-index: 999; /* Behind modal */ }

Position Values Comparison

Position Positioned Relative To Document Flow
static N/A (default) Normal
relative Normal position Normal
absolute Positioned ancestor Removed
fixed Viewport Removed
sticky Viewport + parent Normal (until sticky)

Common Positioning Patterns

Center a Div Absolutely

.centered { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); }

Overlay One Element Over Another

.image-container { position: relative; } .overlay { position: absolute; top: 0; left: 0; width: 100%; height: 100%; background: rgba(0,0,0,0.5); }

Browser Support

All position values are supported in all modern browsers. Sticky positioning has excellent support except in older browsers like Internet Explorer 11.
