Semantic HTML

Lesson 10 of 15

Semantic HTML uses meaningful tags that describe content purpose, not just presentation. Instead of generic <div> elements, semantic tags like <header>, <nav>, <main>, and <footer> tell browsers and assistive technologies what content represents.

What is Semantic HTML?

Semantic HTML means using HTML tags that convey meaning about content. A semantic element clearly describes its meaning to both the browser and developer. Semantic markup improves accessibility, SEO, and code maintainability.

Non-Semantic vs Semantic:

  • Non-Semantic: <div>, <span> – generic containers
  • Semantic: <header>, <nav>, <article> – meaningful structure

Major Semantic Elements

<header>

Top section: site logo, navigation, introductory content

<nav>

Navigation links (primary, secondary)

<main>

Primary content of the page (one per page)

<article>

Self-contained content (blog post, news, comment)

<section>

Thematic grouping of content

<aside>

Sidebar, related links, tangential content

<footer>

Bottom: copyright, links, contact info

<figure>

Self-contained illustration with caption

Semantic Page Structure

<header>
  <h1>Website Title</h1>
  <nav> ... </nav>
</header>

<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>
</main>

<footer>
  <p>Copyright info</p>
</footer>

When to Use Which Element

<header> vs <footer>:

  • <header>: Introductory content at top (can appear in article, section)
  • <footer>: Footer content at bottom (can appear in article, section)

<article> vs <section>:

  • <article>: Independent content (blog post, news article, comment)
  • <section>: Thematic grouping (chapters, tabbed content, related topics)

<main> Rules:

  • Only ONE <main> element per page
  • Should not be inside <article>, <aside>, <footer>, or <header>
  • Contains the unique content of the page

Semantic Benefits

  • Accessibility: Screen readers understand page structure better
  • SEO: Search engines understand content hierarchy
  • Maintainability: Code is clearer and easier to understand
  • Standards Compliance: Follows HTML5 best practices
  • Responsive Design: Easier to style for different layouts

Non-Semantic Page Structure (DON'T DO THIS)

<div class="header">
  <h1>Website Title</h1>
  <div class="nav"> ... </div>
</div>

<div class="main">
  <div class="article">
    <h2>Article Title</h2>
  </div>
</div>

Best Practices with Semantic HTML

✓ DO
  • ✓ Use semantic elements
  • ✓ Structure content logically
  • ✓ One <main> per page
  • ✓ Use <nav> for navigation
  • ✓ Group related content with <section>
✗ DON'T
  • ✗ Use <div> for everything
  • ✗ Multiple <main> elements
  • ✗ Nest incompatible elements
  • ✗ Use <section> as wrapper
  • ✗ Ignore heading hierarchy

Real-World Semantic Example

<!DOCTYPE html>
<html>
<body>
<header><h1>My Blog</h1></header>
<nav><a href="/">Home</a></nav>
<main>
  <article>
    <header><h2>Post Title</h2></header>
    <p>Post content...</p>
    <footer>By Author</footer>
  </article>
</main>
<footer>Copyright</footer>
</body>

Why Semantic HTML Matters

Understanding semantic HTML is crucial because: Semantic HTML creates structured, meaningful content that benefits everyone. It improves accessibility for users with disabilities, helps search engines understand content, and makes code more maintainable. Semantic markup is a core principle of modern web development.
Key Takeaways
  • Semantic HTML uses meaningful tags that describe content purpose
  • Key elements: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>
  • Only one <main> element per page
  • <article> is for independent, self-contained content
  • <section> is for thematic grouping of content
  • Semantic markup improves accessibility and SEO
  • Makes code more maintainable and readable
  • Prefer semantic elements over generic <div> tags
