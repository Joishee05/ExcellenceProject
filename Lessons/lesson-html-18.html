Practical HTMX Examples

Lesson 18 of 18 • Final Lesson

Now that you understand HTMX fundamentals, let's explore real-world examples you can use in the Digital Bones Box project.

Real-World Example 1: Live Search

A search box that shows results as the user types:

<!-- HTML --> <input type="text" name="search" placeholder="Search for items..." hx-get="/search" hx-trigger="keyup changed delay:500ms" hx-target="#search-results"> <div id="search-results"></div>

How it works:

  • User types in the search box
  • After 500ms of no typing (debounce), HTMX sends a GET request to /search with the input value
  • Server responds with HTML snippets of matching results
  • Results appear instantly in the #search-results div

Real-World Example 2: Todo List with Add/Delete

<form hx-post="/todos" hx-target="#todo-list" hx-swap="beforeend"> <input type="text" name="task" placeholder="Add a new task..."> <button type="submit">Add</button> </form> <ul id="todo-list"> <li id="todo-1"> Buy groceries <button hx-delete="/todos/1" hx-target="closest li" hx-swap="outerHTML swap:1s"> Delete </button> </li> <li id="todo-2"> Finish project <button hx-delete="/todos/2" hx-target="closest li" hx-swap="outerHTML swap:1s"> Delete </button> </li> </ul>

Features:

  • Form submits to /todos endpoint using POST
  • Server responds with new <li> element
  • New item is appended to the list (hx-swap="beforeend")
  • Delete button removes the specific todo item
  • hx-swap="outerHTML swap:1s" adds a 1 second swap transition

Real-World Example 3: Infinite Scroll / Load More

<div id="posts"> <!-- Existing posts --> </div> <button hx-get="/posts?page=2" hx-target="#posts" hx-swap="beforeend" hx-indicator=".spinner"> Load More </button> <div class="spinner" style="display:none"> Loading... </div>

Features:

  • Button fetches more posts from the server
  • Posts are appended to the existing list (hx-swap="beforeend")
  • hx-indicator shows a loading spinner while fetching

Real-World Example 4: Form Validation

<form hx-post="/validate-email"> <input type="email" name="email" hx-post="/validate-email" hx-trigger="change" hx-target=".email-error"> <span class="email-error"></span> <button type="submit">Submit</button> </form>

Server Response (when invalid):

<span class="email-error error"> Invalid email format </span>

Server Response (when valid):

<span class="email-error"></span>

Real-World Example 5: Sorting & Filtering

<div hx-target="this" hx-swap="outerHTML"> <select name="sort" hx-get="/items" hx-trigger="change"> <option value="name">Sort by Name</option> <option value="date">Sort by Date</option> <option value="price">Sort by Price</option> </select> <select name="category" hx-get="/items" hx-trigger="change"> <option value="">All Categories</option> <option value="electronics">Electronics</option> <option value="books">Books</option> <option value="clothing">Clothing</option> </select> </div> <div id="items"> <!-- Items will display here --> </div>

Best Practices for HTMX

1. Keep Server Responses Lightweight

Server should respond with HTML fragments only, not full pages:

<!-- Good: Just the fragment --> <li class="todo-item"> <span>Buy groceries</span> <button hx-delete="/todos/1">Delete</button> </li> <!-- ✗ Bad: Full page HTML --> <!DOCTYPE html> <html> <head>...</head> <body>...</body> </html>

2. Use Semantic HTML

HTMX works best with semantic, well-structured HTML. Use proper form elements, buttons, and links.

3. Handle Errors Gracefully

<button hx-post="/action" hx-confirm="Are you sure?" hx-target="#result"> Perform Action </button>

4. Use hx-confirm for Destructive Actions

Always ask for confirmation before deleting or making significant changes.

5. Debounce Frequent Requests

<input hx-get="/search" hx-trigger="keyup changed delay:500ms">

This prevents sending a request for every keystroke, improving performance.

Integrating with Digital Bones Box Project

You can use HTMX to:

  • Live search: Search for students, courses, or resources without page reload
  • Real-time updates: Update grades, attendance, or status instantly
  • Form validation: Validate user input as they type
  • Dynamic filtering: Filter results by date, category, status, etc.
  • Bulk operations: Perform actions on multiple items at once
  • Inline editing: Edit content directly in the table/list
Key Takeaways
  • HTMX shines for live search, load more, and real-time updates
  • Always return HTML fragments from the server, not full pages
  • Use hx-confirm for destructive operations
  • Debounce frequent requests to improve performance
  • HTMX works perfectly with server-side rendering
  • Combine HTMX with your backend to create powerful, interactive experiences
