Lesson 3: Colors & Fonts

Lesson 3 of 15

What You'll Learn

In this comprehensive lesson, you will master working with colors and fonts in CSS. You will learn different color systems, how to choose and implement fonts effectively, and best practices for typography that improves readability and user experience.

Understanding Colors in CSS

Colors are fundamental to web design. CSS offers multiple ways to specify colors, each with its own advantages and use cases.

Color System 1: Named Colors

CSS supports 147 predefined color names. These are easy to remember and use but limit your design flexibility.

color: red; color: blue; color: green; color: lightblue; background: transparent; /* Special keyword */

What This Does:

  • color: red; - Browser recognizes "red" as a color name and displays red text. Simple and readable
  • color: lightblue; - Named colors can have modifiers like "light" or "dark" (e.g., lightblue, darkgreen)
  • background: transparent; - A special keyword that makes something invisible (see-through)
  • Limitation: Only 147 names available. Professional designs need more color options
  • When to use: Quick prototyping or when exact color doesn't matter much

Named colors are useful for quick prototyping but professional designs typically use other color systems for more precise control.

Color System 2: Hexadecimal (Hex) Colors

Hex colors are six-digit codes (or three-digit shorthand) representing RGB values in base-16 format. This is the most common color system in web design.

color: #FF0000; /* Red (full notation) */ color: #F00; /* Red (shorthand) */ color: #00FF00; /* Green */ color: #0000FF; /* Blue */ color: #333333; /* Dark gray */ color: #FFFFFF; /* White */

What This Does:

  • #FF0000: The # symbol means "hex color". FF=red, 00=green, 00=blue. Together they make pure red
  • #F00: Shorthand for #FF0000. The browser automatically expands it. Both create the same red
  • #333333 (dark gray): When all three pairs are equal, you get a shade of gray. 333 is dark, FFF is white, 000 is black
  • Why popular: Designers can create millions of colors. Tools let you pick a color visually and get the hex code
  • Real example: #10b981 means red(10) + green(b9) + blue(81) = a nice teal color

Hex colors range from #000000 (black) to #FFFFFF (white). Every two digits represent Red, Green, and Blue values (0-255). The shorthand (#ABC) expands to #AABBCC.

Color System 3: RGB (Red, Green, Blue)

RGB uses three numbers (0-255) to specify the intensity of red, green, and blue light. This is how monitors display colors.

color: rgb(255, 0, 0); /* Red */ color: rgb(0, 255, 0); /* Green */ color: rgb(0, 0, 255); /* Blue */ color: rgb(128, 128, 128); /* Gray */

What This Does:

  • rgb(255, 0, 0): Three numbers from 0-255. First is red, second is green, third is blue. This creates pure red
  • rgb(0, 255, 0): Red=0 (no red), Green=255 (maximum), Blue=0 (no blue). This makes pure green
  • rgb(128, 128, 128): When all three are equal, you get gray. 0,0,0 = black, 255,255,255 = white
  • How monitors work: Every pixel on your screen is made of tiny red, green, and blue lights. RGB controls their brightness
  • More intuitive: If you know "I want more red", RGB is easier than remembering hex codes

Color System 4: RGBA (RGB with Alpha/Transparency)

RGBA adds an alpha channel for transparency. The fourth number ranges from 0 (fully transparent) to 1 (fully opaque).

background: rgba(255, 0, 0, 1); /* Fully opaque red */ background: rgba(255, 0, 0, 0.5); /* 50% transparent red */ background: rgba(0, 0, 0, 0.8); /* 80% opaque black */ background: rgba(255, 255, 255, 0); /* Fully transparent */

What This Does:

  • rgba(255, 0, 0, 1): Same as rgb but with a 4th number. The 1 means 100% opaque (solid, can't see through)
  • rgba(255, 0, 0, 0.5): The 0.5 means 50% transparent (semi-transparent). You can see through it
  • The Alpha number: 0 = completely invisible, 0.5 = halfway see-through, 1 = completely solid
  • Practical use: Semi-transparent overlays on images, buttons that fade in/out, glass-morphism effects
  • rgba(255, 255, 255, 0): White that's completely transparent - totally invisible but still takes up space

Color System 5: HSL (Hue, Saturation, Lightness)

HSL is another way to specify colors. It's often more intuitive for designers: Hue (0-360 degrees), Saturation (0-100%), Lightness (0-100%).

color: hsl(0, 100%, 50%); /* Red */ color: hsl(120, 100%, 50%); /* Green */ color: hsl(240, 100%, 50%); /* Blue */ color: hsl(200, 50%, 50%); /* Medium blue */

What This Does:

  • Hue (0-360): Think of a color wheel. 0=red, 120=green, 240=blue. Any angle gives you a different color
  • Saturation (0-100%): How intense the color is. 0% = gray, 100% = full, vibrant color
  • Lightness (0-100%): How bright. 0% = black, 50% = normal, 100% = white
  • Real example: hsl(200, 50%, 50%) = blue-ish (200 degrees) + medium intensity (50% saturation) + medium brightness (50% light)
  • Why designers love it: To make a color darker, just lower lightness. To make it less vibrant, lower saturation. Much easier than hex!

HSL can be easier for creating color schemes and tints/shades of the same color.

Color System 6: HSLA (HSL with Alpha)

HSLA combines HSL with transparency, similar to RGBA.

background: hsla(200, 100%, 50%, 0.5); /* Semi-transparent light blue */

What This Does:

  • hsla(200, 100%, 50%, 0.5): First three numbers are HSL (hue, saturation, lightness), fourth is transparency
  • Advantages: Combine the easy HSL system with transparency
  • The 0.5: 50% transparent - you can see through it

Working with Color Effectively

Color Properties

color: #333; /* Text color */ background-color: #f5f5f5; /* Background color */ border-color: #ddd; /* Border color */ box-shadow: 0 2px 5px rgba(0,0,0,0.2); /* Shadow color with transparency */ text-shadow: 2px 2px 4px rgba(0,0,0,0.3); /* Text shadow */

Color Contrast and Accessibility

Proper color contrast is essential for accessibility. The Web Content Accessibility Guidelines (WCAG) recommend:

  • WCAG AA: Contrast ratio of at least 4.5:1 for normal text
  • WCAG AAA: Contrast ratio of at least 7:1 for normal text

Always test your color combinations to ensure they are readable by people with color blindness and low vision.

Introduction to Fonts and Typography

Typography is crucial for user experience. Good typography improves readability, establishes visual hierarchy, and reinforces your brand identity.

Font Families

The font-family property specifies which font to use. You can list multiple fonts as fallbacksthe browser uses the first available font.

/* Using specific fonts with fallbacks */ font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif; /* Alternative approaches */ font-family: 'Times New Roman', Georgia, serif; font-family: 'Courier New', monospace; font-family: Arial, Helvetica, sans-serif;

Font Categories

  • Serif: Fonts with small lines at the end of letters (e.g., Times New Roman, Georgia). Often used for body text in print but increasingly common online.
  • Sans-Serif: Fonts without serifs (e.g., Arial, Helvetica, Segoe UI). Cleaner appearance, preferred for web and screen readability.
  • Monospace: Fixed-width fonts where each letter takes same space (e.g., Courier New, Monaco). Used for code blocks.
  • Cursive: Script or handwriting-style fonts. Use sparingly as they can reduce readability.
  • Fantasy: Decorative fonts. Use only for headings and special effects.

Font Properties

Font Size

font-size: 16px; /* Pixels (fixed size) */ font-size: 1.5em; /* Em (relative to parent) */ font-size: 1.5rem; /* Rem (relative to root) */ font-size: 100%; /* Percentage of parent */

Font Weight

Controls how thick or thin the font appears. Values range from 100 (thin) to 900 (black).

font-weight: normal; /* 400 */ font-weight: bold; /* 700 */ font-weight: 300; /* Light */ font-weight: 500; /* Medium */ font-weight: 700; /* Bold */ font-weight: 900; /* Black */

Font Style

font-style: normal; /* Default */ font-style: italic; /* Italicized text */ font-style: oblique; /* Similar to italic */

Line Height

Controls the vertical spacing between lines of text. Affects readability significantly.

line-height: 1.5; /* 1.5 times the font size (recommended for web) */ line-height: 1.6; /* Even more space */ line-height: 20px; /* Fixed pixel height */ line-height: 150%; /* Percentage of font size */

For body text, a line-height of 1.5 to 1.8 is recommended for optimal readability.

Typography Best Practices

  • Font Size: Use 16px or larger for body text. Smaller text reduces readability.
  • Line Height: Use 1.5-1.8 for body text. Too tight reduces readability, too loose looks awkward.
  • Font Families: Limit to 2-3 font families per site. Too many fonts look unprofessional.
  • Font Pairing: Pair contrasting fonts well (e.g., sans-serif heading with serif body, or vice versa).
  • Color Contrast: Ensure sufficient contrast between text and background.
  • Letter Spacing: Use sparingly. Excessive letter spacing reduces readability.

Web Fonts and @font-face

Beyond system fonts, you can use custom web fonts via @font-face or services like Google Fonts.

@font-face { font-family: 'MyCustomFont'; src: url('font.woff2') format('woff2'); } body { font-family: 'MyCustomFont', sans-serif; }

Text Properties Beyond Font

text-align: left; /* left, center, right, justify */ text-decoration: none; /* none, underline, overline, line-through */ text-transform: uppercase; /* uppercase, lowercase, capitalize, none */ text-indent: 20px; /* Indent first line of paragraph */ letter-spacing: 2px; /* Space between letters */ word-spacing: 5px; /* Space between words */

Creating Readable Content

  • Font Size: 16px minimum for body text
  • Line Height: 1.5-1.8 for readable spacing
  • Line Length: 50-75 characters per line for optimal readability
  • Color Contrast: High contrast between text and background
  • Font Weight: Use bold for emphasis, not for large text areas
