Lesson 2: Selectors & Properties

Lesson 2 of 15

What You'll Learn

In this comprehensive lesson, you will master CSS selectors and propertiesthe foundation of all CSS styling. You will learn different selector types, how to target specific elements, the most important CSS properties, and how to organize your styles effectively.

Introduction to Selectors

Selectors are patterns used to select HTML elements that you want to style. Every CSS rule starts with a selector that determines which elements the styles will apply to. Understanding selectors is crucial for writing effective CSS.

Types of CSS Selectors

1. Element/Type Selector

Targets all instances of a specific HTML element by its tag name.

p { color: #333; } /* Styles all paragraphs */ h1 { font-size: 32px; } /* Styles all h1 headings */ div { background: white; } /* Styles all divs */

Breaking Down Each Line:

  • p { } - The "p" targets all paragraph tags. Every <p></p> on your page gets the same styling
  • color: #333; - Changes text color to dark gray (#333 is a hex code for gray)
  • h1 { font-size: 32px; } - Makes all headings 32 pixels tall. That's pretty big!
  • Why use type selectors?: When you want to style ALL of one type of element the same way (like all paragraphs)

2. Class Selector

Targets elements with a specific class attribute. Class selectors start with a dot (.) and can be applied to multiple elements.

.intro { font-weight: bold; } /* Any element with class="intro" */ .button { background: #0066cc; } /* Any element with class="button" */ <!-- In your HTML --> <p class="intro">This is an intro paragraph</p> <button class="button">Click Me</button>

What This Does:

  • .intro { } - The dot "." means "look for elements with class='intro'". Only elements with this class get styled
  • font-weight: bold; - Makes the text bold/thick
  • Key Difference: Type selectors (like "p") style ALL paragraphs. Class selectors (like ".intro") style ONLY paragraphs with class="intro"
  • Why useful: You can have multiple paragraphs but only make SOME of them bold by using the class
  • Reusable: You can add class="intro" to any element - not just paragraphs. It always gets bold styling

3. ID Selector

Targets a single element with a specific ID. ID selectors start with a hash (#) and should be unique on the page. Use IDs sparingly in CSSclasses are more flexible.

#header { background: #333; } /* The element with id="header" */ #main { max-width: 1200px; } /* The element with id="main" */

What This Does:

  • #header { } - The hash "#" means "find the ONE element with id='header'". Only ONE element should have a specific ID
  • background: #333; - Sets the background color to dark gray (#333)
  • Classes vs IDs: Classes (.) can be used multiple times. IDs (#) should be used only once
  • When to use: Use IDs for unique page sections like header, footer, main content - the big landmark elements

4. Descendant Selector

Targets elements that are descendants of another element, regardless of depth. Useful for styling elements within specific containers.

/* Style all li elements inside a ul */ ul li { list-style: none; } /* Style all p elements inside .container */ .container p { color: #666; }

What This Does:

  • ul li { } - The space means "li that is INSIDE ul". Styles list items only when they're in a ul (not outside it)
  • list-style: none; - Removes the bullets from list items
  • .container p { } - Targets paragraphs ONLY if they're inside something with class="container"
  • Why useful: You can have paragraphs outside .container (not styled) and inside .container (styled gray). Same HTML tag, different styling based on location!
  • Deep nesting: The p can be deeply nested inside .container - 2 levels down, 5 levels down, doesn't matter

5. Child Selector

Targets elements that are direct children of another element (one level deep only). Uses the greater-than symbol (>).

/* Style only direct child li elements, not nested ones */ ul > li { margin-bottom: 10px; } /* Style only direct child paragraphs */ .section > p { font-size: 16px; }

What This Does:

  • ul > li { } - The ">" means "direct child". Styles li ONLY if it's immediately inside ul (not nested deeper)
  • Difference from descendant: "ul li" (space) = any li anywhere inside. "ul > li" (>) = only li directly inside
  • margin-bottom: 10px; - Adds 10 pixels of space below each list item
  • Why useful: More precise control. You can style direct children differently from nested descendants

6. Attribute Selector

Targets elements based on their attributes. Very useful for styling form inputs and links.

/* Style all input elements with type="text" */ input[type="text"] { border: 1px solid #ccc; padding: 8px; } /* Style all links with target="_blank" */ a[target="_blank"] { color: #0066cc; } /* Style elements with specific attribute values */ img[alt] { border: 1px solid #ddd; }

What This Does:

  • input[type="text"] { } - The brackets [ ] mean "check the attribute". Styles only input elements WITH type="text" attribute
  • border: 1px solid #ccc; - Adds a 1-pixel gray border. padding: 8px adds space inside
  • a[target="_blank"] { } - Styles only links that have target="_blank" (opens in new tab)
  • img[alt] { } - Styles images that HAVE an alt attribute (whether value matters or not)
  • Why useful: You can style inputs differently based on their type. Text inputs look different from checkboxes

7. Pseudo-Class Selector

Targets elements based on their state or position. Common pseudo-classes include :hover, :focus, :first-child, :last-child.

/* Style links when hovered */ a:hover { color: #0052a3; text-decoration: underline; } /* Style focused form inputs */ input:focus { outline: 2px solid #0066cc; } /* Style first child of a parent */ li:first-child { font-weight: bold; } /* Style odd/even list items */ li:nth-child(odd) { background: #f5f5f5; }

What This Does:

  • a:hover { } - The colon ":" means "this is a pseudo-class". Styles links when mouse hovers over them. Makes them darker and underlined
  • input:focus { } - Styles input fields when they're active (user clicked in them to type). Shows a blue outline
  • li:first-child { } - Styles only the FIRST list item. Makes it bold
  • li:nth-child(odd) { } - Styles every odd-numbered list item (1st, 3rd, 5th...). Gives them light gray background
  • Why useful: Pseudo-classes let you style based on user interaction (hover, focus) or position (first, last, odd)

8. Combining Selectors

You can combine multiple selectors for more specific targeting. This is called using multiple selectors.

/* Style all h1, h2, h3 elements */ h1, h2, h3 { font-family: 'Segoe UI', sans-serif; color: #333; } /* Style paragraphs inside containers with the .content class */ .content p { line-height: 1.8; } /* Style the first paragraph inside a .section */ .section p:first-child { margin-top: 0; }

What This Does:

  • h1, h2, h3 { } - The comma "," means "apply these styles to ALL these selectors". Saves writing three separate rules
  • .content p { } - Combines class and descendant selector. Styles paragraphs ONLY inside .content
  • .section p:first-child { } - Combines class, descendant, and pseudo-class. Styles the FIRST paragraph inside .section
  • Why useful: More specific control. You can style the same element differently in different locations

Specificity: Which Rule Wins?

When multiple CSS rules target the same element, the browser uses specificity to determine which rule applies. Specificity is calculated as follows:

  • Element/Type Selector: 1 point (lowest specificity)
  • Class Selector (.class): 10 points
  • ID Selector (#id): 100 points
  • Inline Style: 1000 points (highest)
/* Example: Which rule applies? */ p { color: black; } /* Specificity: 1 */ .intro { color: blue; } /* Specificity: 10 (wins over p) */ #main p { color: red; } /* Specificity: 101 (wins over .intro) */ <p style="color: green;"> /* Specificity: 1000 (highest, wins) */

What This Means:

  • Specificity: A score that determines which CSS rule applies when multiple rules target the same element
  • Type selector (1 point): "p { }" is weak. Any other rule wins
  • Class selector (10 points): ".intro { }" beats type selectors. Stronger than just "p"
  • ID selector (100 points): "#main p { }" is powerful. Beats classes and type selectors
  • Inline style (1000 points): "style=" in HTML is the strongest. Beats everything else
  • Real Example: If a paragraph has both "p { color: black; }" AND ".intro { color: blue; }", it becomes BLUE because class (10) beats type (1)

Common CSS Properties

Text and Font Properties

color: #333; /* Text color */ font-size: 16px; /* Text size */ font-weight: bold; /* Text weight (normal, bold, 700, etc.) */ font-family: Arial, sans-serif; /* Font choice */ text-align: center; /* Text alignment (left, center, right, justify) */ line-height: 1.6; /* Line spacing */ text-decoration: none; /* Text decoration (underline, overline, line-through) */ letter-spacing: 2px; /* Space between letters */

Color and Background Properties

color: #333; /* Text color (hexadecimal) */ color: rgb(51, 51, 51); /* Text color (RGB) */ color: rgba(51, 51, 51, 0.8); /* Text color with transparency */ background-color: #f5f5f5; /* Background color */ background-image: url('image.jpg'); /* Background image */ background-size: cover; /* How to size the background */ background-position: center; /* Where to position the background */

Box Model Properties

margin: 20px; /* Space outside element */ padding: 15px; /* Space inside element */ border: 1px solid #ddd; /* Border around element */ width: 100%; /* Element width */ height: auto; /* Element height */ max-width: 1200px; /* Maximum width */ min-height: 300px; /* Minimum height */

Display and Layout Properties

display: block; /* Block-level element (full width) */ display: inline; /* Inline element (flows with text) */ display: inline-block; /* Both inline and block properties */ display: flex; /* Flexbox container */ display: grid; /* Grid container */ display: none; /* Hide element completely */

CSS Property Categories

  • Typography: font-size, font-weight, font-family, color, line-height, text-align
  • Spacing: margin, padding, gap
  • Borders and Backgrounds: border, background-color, background-image, box-shadow
  • Layout: display, width, height, position, float
  • Visual Effects: opacity, transform, filter, transition
  • Text Effects: text-decoration, text-transform, text-shadow

Inheritance in CSS

Some CSS properties are inherited from parent elements to their children. For example, the color property is inherited, so if you set color on a parent div, child elements will have that color too.

/* Inherited property example */ .container { color: #333; } /* All text inside .container will be this color */ /* Non-inherited property example */ .container { border: 1px solid #ddd; } /* Only .container gets the border, not children */

You can force inheritance using the inherit keyword, or reset inheritance using the initial keyword.

Shorthand Properties

Some CSS properties have shorthand versions that let you set multiple values at once. This reduces code and improves readability.

/* Longhand */ margin-top: 10px; margin-right: 20px; margin-bottom: 10px; margin-left: 20px; /* Shorthand (same effect) */ margin: 10px 20px 10px 20px; /* top right bottom left */ /* Even shorter */ margin: 10px 20px; /* top/bottom right/left */

Debugging CSS Issues

Every modern browser has developer tools to help debug CSS issues:

  • Inspect Element: Right-click any element and select "Inspect" to see its HTML and CSS
  • Box Model Visualization: The Elements panel shows how margin, border, padding, and content are sized
  • Computed Styles: See which CSS rules actually apply to an element
  • CSS Override Visualization: Crossed-out rules show which CSS was overridden and why

Best Practices for Writing Selectors

  • Keep Specificity Low: Avoid deeply nested selectors and ID-based selectors
  • Use Meaningful Class Names: Name classes based on purpose (e.g., .button-primary), not appearance (e.g., .blue-button)
  • Avoid Inline Styles: Always use external or internal stylesheets instead of inline styles
  • Be Consistent: Use a consistent naming convention (e.g., kebab-case for class names)
  • Don't Overuse !important: Only use !important as a last resort
