Tables

Lesson 8 of 15

Tables organize data into rows and columns for easy comparison and analysis. HTML tables use semantic elements like <table>, <thead>, <tbody>, and <tfoot> to structure tabular data clearly and accessibly.

What Are Tables?

A table is a structured grid of data organized in rows and columns. Tables are essential for presenting data like schedules, pricing, statistics, and comparisons. Proper table markup improves accessibility for screen readers and makes data easier to understand.

When to Use Tables:

  • Price comparison charts
  • Sports scores and statistics
  • Class schedules or timetables
  • Financial reports or data
  • Feature comparisons

Basic Table Structure

Every table consists of key elements: <table> (wrapper), <tr> (table row), and <td> (table data cell). Header rows use <th> (table header) instead of <td>.

Simplest Table Example:

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>Alice</td>
    <td>28</td>
  </tr>
</table>

Table Elements Explained

<table>

Purpose: Container for entire table

The wrapper element that contains all rows, headers, and data

<tr>

Purpose: Table row – groups cells horizontally

Contains multiple <th> or <td> elements

<th>

Purpose: Table header cell (semantic)

Bold by default; helps screen readers understand columns

<td>

Purpose: Table data cell

Regular cell containing data; used with rows

Semantic Table Sections

Use <thead>, <tbody>, and <tfoot> to organize tables semantically and improve accessibility:

Complete Table with Sections:

<table>
  <thead>
    <tr>
      <th>Product</th>
      <th>Price</th>
      <th>Stock</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Laptop</td>
      <td>$999</td>
      <td>15</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="2">Total:</td>
      <td>15 items</td>
    </tr>
  </tfoot>
</table>

Understanding Each Section:

  • <thead>: Table header – typically contains column titles in <th> elements
  • <tbody>: Table body – contains main data rows
  • <tfoot>: Table footer – contains summary rows (totals, averages, notes)

Important Table Attributes

colspan - Span Multiple Columns:

<!-- Cell spans 2 columns -->
<td colspan="2">Merged cell</td>

<!-- Header spanning 3 columns -->
<th colspan="3">Q1 Sales Data</th>

rowspan - Span Multiple Rows:

<!-- Cell spans 2 rows -->
<td rowspan="2">Name</td>

<!-- Useful for labels in side columns -->
<th rowspan="3">2024</th>

Table Best Practices

✓ DO
  • ✓ Use <thead>, <tbody>, <tfoot>
  • ✓ Use <th> for headers
  • ✓ Keep headers semantic
  • ✓ Use <caption> for table title
  • ✓ Add scope attribute to headers
✗ DON'T
  • ✗ Use tables for page layout
  • ✗ Mix <th> and <td> in headers
  • ✗ Skip semantic sections
  • ✗ Overuse colspan/rowspan
  • ✗ Create complex nested tables

Accessible Tables

Screen reader users need proper semantic markup to navigate tables. Here are accessibility best practices:

Add Scope Attribute to Headers:

<table>
  <thead>
    <tr>
      <th scope="col">Name</th><!-- column header -->
      <th scope="col">Age</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Alice</th><!-- row header -->
      <td>28</td>
    </tr>
  </tbody>
</table>

Add Caption:

<table>
  <caption>2024 Sales by Region</caption>
  <thead> ... </thead>
</table>

Real-World Table Example

<table>
  <caption>Course Schedule Spring 2024</caption>
  <thead>
    <tr>
      <th scope="col">Course</th>
      <th scope="col">Time</th>
      <th scope="col">Instructor</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>HTML 101</td>
      <td>9:00 AM</td>
      <td>John Smith</td>
    </tr>
  </tbody>
</table>

Why Tables Matter

Understanding tables is crucial because: Tables are essential for displaying structured data on the web. Proper table markup using semantic elements ensures data is accessible to all users, including those using assistive technologies. Well-structured tables improve readability, maintainability, and SEO while maintaining proper document semantics.
Key Takeaways
  • Tables organize data in rows and columns using <tr>, <th>, and <td>
  • Use <thead>, <tbody>, and <tfoot> for semantic structure
  • Use <th> for headers, not <td>
  • colspan and rowspan allow cells to span multiple columns or rows
  • Add scope attribute to headers for accessibility
  • Use <caption> to provide table title
  • Never use tables for page layout
  • Proper semantics improve screen reader accessibility
