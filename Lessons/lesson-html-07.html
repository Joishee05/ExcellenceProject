Lists

Lesson 7 of 15

Lists organize related content in a clear, scannable format. HTML provides two main list types: unordered lists (with bullets) and ordered lists (with numbers). Lists are fundamental for navigation menus, to-do items, instructions, and any grouped content.

What Are Lists?

A list is a set of related items grouped together. Lists help organize information so it's easy to read and understand. They're used everywhere on the web: navigation menus, product features, numbered steps, and more.

Why Use Lists?

  • Improves readability and scannability
  • Semantically meaningful to screen readers
  • Easy to style with CSS
  • Standard way to group related content

Unordered Lists (<ul>)

Unordered lists display items with bullet points. Use <ul> (unordered list) for items where order doesn't matter. Each item goes in an <li> (list item) tag.

Basic Unordered List Syntax:

<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Juice</li>
</ul>

When to Use Unordered Lists:

  • Navigation menus (site sections, links)
  • Product features or benefits
  • Team member names
  • Any items without a specific order

Ordered Lists (<ol>)

Ordered lists display items with numbers or letters. Use <ol> (ordered list) when the sequence matters, like steps in a recipe or ranking items by importance.

Basic Ordered List Syntax:

<ol>
  <li>Preheat oven to 350°F</li>
  <li>Mix ingredients together</li>
  <li>Pour into baking pan</li>
  <li>Bake for 30 minutes</li>
</ol>

When to Use Ordered Lists:

  • Step-by-step instructions
  • Ranking (1st place, 2nd place, etc.)
  • Numbered procedures or processes
  • Any items requiring a specific sequence

Nested Lists

Lists can contain other lists inside them. A nested list is a list inside another list item. This is useful for showing hierarchical relationships or subcategories.

Nested List Example:

<ul>
  <li>Fruits
    <ul>
      <li>Apples</li>
      <li>Bananas</li>
    </ul>
  </li>
  <li>Vegetables
    <ul>
      <li>Carrots</li>
      <li>Celery</li>
    </ul>
  </li>
</ul>

Important: The nested list must be placed inside an <li> tag, not directly inside the parent <ul> or <ol> tag.

List Type Comparison

Unordered Lists (<ul>)
  • Uses bullet points
  • Order doesn't matter
  • Common for navigation
  • Default bullet style can be changed with CSS

Best for: Features, benefits, categories, links

Ordered Lists (<ol>)
  • Uses numbers (1, 2, 3...)
  • Order matters
  • Common for instructions
  • Can use letters (a, b, c...) or roman numerals (i, ii, iii...)

Best for: Steps, rankings, procedures, sequences

List Attributes

The <ol> tag supports attributes to control numbering style:

Type Attribute - Changes Number Format:

<!-- Default: numbers (1, 2, 3...) -->
<ol type="1"><li>First</li></ol>

<!-- Lowercase letters (a, b, c...) -->
<ol type="a"><li>First</li></ol>

<!-- Uppercase letters (A, B, C...) -->
<ol type="A"><li>First</li></ol>

<!-- Lowercase roman numerals (i, ii, iii...) -->
<ol type="i"><li>First</li></ol>

<!-- Uppercase roman numerals (I, II, III...) -->
<ol type="I"><li>First</li></ol>

Start Attribute - Begins Numbering at Specific Number:

<ol start="5">
  <li>Fifth item</li>
  <li>Sixth item</li>
  <li>Seventh item</li>
</ol>

List Best Practices

✓ DO
  • ✓ Use semantic list elements (ul, ol, li)
  • ✓ Use <ol> for sequences and rankings
  • ✓ Use <ul> for unordered items
  • ✓ Keep list items concise and clear
  • ✓ Use nested lists for hierarchies
✗ DON'T
  • ✗ Use <br> tags to create lists
  • ✗ Use <ol> for items without order
  • ✗ Place nested lists outside <li> tags
  • ✗ Skip <li> tags inside <ul> or <ol>
  • ✗ Create overly long list items

Real-World List Examples

Navigation Menu (Unordered List):

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/services">Services</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

Recipe Instructions (Ordered List):

<h2>How to Make Chocolate Cake</h2>
<ol>
  <li>Preheat oven to 350°F</li>
  <li>Mix flour, sugar, cocoa powder, and salt</li>
  <li>Add eggs, oil, and vanilla extract</li>
  <li>Pour batter into greased pan</li>
  <li>Bake for 30-35 minutes until done</li>
</ol>

Product Features (Unordered List):

<h3>Premium Headphones Features:</h3>
<ul>
  <li>40-hour battery life</li>
  <li>Active noise cancellation</li>
  <li>Wireless Bluetooth connectivity</li>
  <li>Premium sound quality</li>
</ul>

Why Lists Matter

Understanding lists is crucial because: Lists are one of the most fundamental HTML elements. They appear in navigation menus, instructions, feature lists, and countless other places on the web. Using semantic list markup improves accessibility for screen readers, enhances SEO, and makes your content easier to scan and understand. Lists demonstrate your understanding of semantic HTML structure.
Key Takeaways
  • Unordered lists (<ul>) use bullet points for unordered items
  • Ordered lists (<ol>) use numbers for sequences and rankings
  • Each list item must be wrapped in <li> tags
  • Nested lists go inside <li> tags of parent lists
  • Use type attribute on <ol> to change numbering (a, A, i, I)
  • Use start attribute on <ol> to begin numbering at specific number
  • Lists are semantic and improve accessibility for screen readers
  • Navigation menus are commonly created with <ul> lists
