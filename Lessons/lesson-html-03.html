Headings & Paragraphs

Lesson 3 of 15

Headings and paragraphs are fundamental building blocks for organizing text content on web pages. They help structure your content in a way that's easy for both users and search engines to understand.

What Are Headings?

Headings are used to create hierarchical structure in your content. HTML provides six levels of headings, from <h1> (most important) to <h6> (least important). Think of headings like an outline in a document – they show the relationships and importance of different sections.

The Heading Hierarchy

Each heading level has a specific purpose and importance level:

H1 - Main Heading

Use for: Page title (should be only ONE per page)

<h1>Welcome to My Website</h1>

Most important heading. Largest and most visually prominent.

H2 - Section Heading

Use for: Major sections of your page

<h2>About Our Service</h2>

Second level of importance. Divides page into major topics.

H3 - Subsection Heading

Use for: Subsections under H2

<h3>Basic Features</h3>

Third level of importance. Breaks down sections further.

H4, H5, H6 - Minor Headings

Use for: Detailed subsections (use sparingly)

<h4>, <h5>, <h6>

Lower importance headings. Use only for very detailed content structure.

Understanding Heading Importance

The heading level you choose affects more than just how the text looks. Headings communicate document structure to search engines, accessibility tools, and assistive technologies used by people with disabilities:

  • Search Engine Optimization: Search engines use heading structure to understand your page's topic and main points
  • Accessibility: Screen readers use headings to let users navigate pages and understand document structure
  • User Experience: Users scan pages looking for headings, so good heading structure improves usability
  • Document Outline: Headings create a logical outline of your content, making it easier to follow

Heading Best Practices

Follow these guidelines when using headings:

✓ DO
  • Use one H1 per page
  • Use headings in order (h1 → h2 → h3)
  • Use descriptive heading text
  • Keep heading text concise
  • Use headings for content structure
✗ DON'T
  • Use multiple H1s on one page
  • Skip heading levels (h1 → h3)
  • Use headings just for styling
  • Use vague heading text
  • Make headings unnecessarily long

What Are Paragraphs?

The <p> tag is used to define a paragraph of text. Browsers automatically add space before and after paragraphs, creating clear separation between text blocks. This makes content easier to read and understand.

Paragraph Example

<h1>Article Title</h1>
<p>This is the introduction paragraph. It introduces the topic and sets the stage for what follows.</p>
<p>This is the second paragraph. Each paragraph is a separate block of text with space around it.</p>
</body>

Proper Heading Structure Example

Here's a real-world example showing proper heading hierarchy on a blog post:

<!DOCTYPE html>
<html>
<head>
  <title>My Blog</title>
</head>
<body>
  <h1>Welcome to My Tech Blog</h1>
  <p>I share articles about web development and coding.</p>

  <h2>Getting Started with HTML</h2>
  <p>HTML is the foundation of all websites...</p>

  <h3>What HTML Does</h3>
  <p>HTML provides structure to web content.</p>

  <h3>Why Learn HTML First</h3>
  <p>Before CSS or JavaScript, you need HTML.</p>

  <h2>CSS for Styling</h2>
  <p>Once you have HTML, CSS makes it look good.</p>
</body>
</html>

Common Heading Mistakes to Avoid

Watch out for these common heading-related mistakes:

  • Multiple H1 tags: Only use one H1 per page – it should represent your page's main topic
  • Skipping levels: Don't jump from H1 directly to H3 – use H2 in between
  • Using headings for styling: Don't use headings just to make text bold or large – use CSS for styling
  • Empty headings: Always include descriptive text in your headings
  • Long headings: Keep headings concise and focused (usually 3-10 words)

Testing Your Heading Structure

To test if your headings are properly structured, try this:

  1. Use a heading hierarchy checker: Browser extensions can visualize your page's heading structure
  2. Read headings in order: Your headings should tell a story about your content
  3. Test with a screen reader: Use accessibility tools to verify heading navigation
  4. Check search engines: Some SEO tools show how search engines see your headings

Why This Matters for Your Learning

Understanding headings and paragraphs is crucial because: They are the most common elements you'll use. Almost every HTML page has multiple headings and paragraphs. Proper heading structure also improves accessibility, making your site usable for people with disabilities and helping search engines understand your content. This is fundamental web development knowledge that will benefit you throughout your career.
Key Takeaways
  • Headings create document structure (H1 is most important, H6 is least)
  • Use only ONE H1 per page for your main topic
  • Use H2 for major sections, H3 for subsections
  • Never skip heading levels (don't jump from H1 to H3)
  • Heading structure helps both accessibility and SEO
  • Paragraphs (<p>) organize text into readable blocks
  • Use headings for content structure, not styling
  • Descriptive, concise headings work best
