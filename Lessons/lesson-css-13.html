Lesson 13: CSS Variables

Lesson 13 of 15

What You'll Learn

In this lesson, you will master CSS variables (also called custom properties)a powerful feature for creating maintainable, flexible stylesheets. Variables let you store values and reuse them throughout your CSS.

Introduction to CSS Variables

CSS variables are reusable values you define once and use multiple times. They improve maintainability and make theme changes simple.

Problem Without Variables

/* Repeating the same color throughout */ .button { color: #10b981; } .link { color: #10b981; } .header { border-color: #10b981; } .badge { background: #10b981; } /* If you need to change the color, you must update it everywhere */

Solution With Variables

/* Define the color ONCE */ :root { --primary-color: #10b981; /* Create a variable storing the color value */ } /* Use the variable EVERYWHERE */ .button { color: var(--primary-color); } /* Insert the variable value here */ .link { color: var(--primary-color); } .header { border-color: var(--primary-color); } .badge { background: var(--primary-color); }

What This Does:

  • --primary-color: A custom CSS variable (starts with --). Stores the green color value
  • :root { }: Makes the variable available everywhere in the page (it's global)
  • var(--primary-color): Inserts the variable value. It's like a placeholder that gets replaced with #10b981
  • BIG ADVANTAGE: Later, if you want to change the color, you change it ONCE in :root and it updates EVERYWHERE automatically!

Real Example: Imagine you use a brand color in 50 places. Without variables, you'd change it 50 times. With variables, you change it once. That's huge for large projects!

Declaring CSS Variables

Global Variables

Variables declared in :root are available everywhere in your stylesheet.

:root { --primary-color: #10b981; --secondary-color: #059669; --background: #f9fafb; --text-color: #1f2937; --spacing: 16px; --border-radius: 8px; }

Local Variables

Variables declared in a specific selector are only available within that scope.

.card { --padding: 20px; --shadow: 0 4px 6px rgba(0,0,0,0.1); padding: var(--padding); box-shadow: var(--shadow); }

Using CSS Variables

Access variables with the var() function.

color: var(--primary-color); background: var(--background); border-radius: var(--border-radius);

Fallback Values

Provide a fallback value if the variable is not defined.

color: var(--primary-color, blue); /* Use blue if --primary-color not defined */ padding: var(--spacing, 16px); /* Use 16px if --spacing not defined */

Common CSS Variables Pattern

:root { /* Colors */ --primary: #10b981; --secondary: #059669; --danger: #ef4444; --warning: #f59e0b; --success: #10b981; --info: #3b82f6; /* Spacing */ --spacing-xs: 4px; --spacing-sm: 8px; --spacing-md: 16px; --spacing-lg: 24px; --spacing-xl: 32px; /* Typography */ --font-primary: "Segoe UI", Tahoma, Geneva, Verdana, sans-serif; --font-monospace: "Courier New", monospace; --font-size-base: 16px; --font-size-sm: 14px; --font-size-lg: 18px; --line-height: 1.6; /* Borders */ --border-radius: 8px; --border-width: 2px; /* Shadows */ --shadow-sm: 0 1px 2px rgba(0,0,0,0.05); --shadow-md: 0 4px 6px rgba(0,0,0,0.1); --shadow-lg: 0 10px 15px rgba(0,0,0,0.1); }

Using Variables in Real Projects

Button Styles Using Variables

:root { --btn-padding: 10px 20px; --btn-border-radius: 4px; --btn-bg: #10b981; --btn-color: white; --btn-hover-bg: #059669; } button { padding: var(--btn-padding); border-radius: var(--btn-border-radius); background: var(--btn-bg); color: var(--btn-color); transition: background 0.3s; } button:hover { background: var(--btn-hover-bg); }

Card Component Variables

:root { --card-padding: var(--spacing-lg); --card-border-radius: 8px; --card-shadow: var(--shadow-md); --card-bg: white; --card-border: 1px solid #e5e7eb; } .card { padding: var(--card-padding); border-radius: var(--card-border-radius); box-shadow: var(--card-shadow); background: var(--card-bg); border: var(--card-border); }

Theme Switching with Variables

Switch themes by updating variable values.

/* Light theme (default) */ :root { --bg-color: white; --text-color: #1f2937; --border-color: #e5e7eb; } /* Dark theme */ :root.dark-mode { --bg-color: #1f2937; --text-color: white; --border-color: #374151; } /* Use throughout stylesheet */ body { background: var(--bg-color); color: var(--text-color); } .card { border-color: var(--border-color); }

JavaScript Theme Switcher

// Toggle dark mode document.documentElement.classList.toggle('dark-mode'); // Or set it dynamically function setTheme(theme) { document.documentElement.classList.remove('light-mode', 'dark-mode'); document.documentElement.classList.add(theme); }

Responsive Variables

Change variable values at different screen sizes.

:root { --font-size: 16px; --spacing: 16px; } @media (max-width: 768px) { :root { --font-size: 14px; --spacing: 12px; } } body { font-size: var(--font-size); } .container { padding: var(--spacing); }

Advanced Variable Techniques

Variables with calc()

:root { --base-spacing: 8px; --double-spacing: calc(var(--base-spacing) * 2); --triple-spacing: calc(var(--base-spacing) * 3); } .small { padding: var(--base-spacing); } .medium { padding: var(--double-spacing); } .large { padding: var(--triple-spacing); }

Conditional Variables with Fallbacks

/* Graceful degradation */ color: var(--text-color, var(--fallback-color, black));

Variable Best Practices

  • Naming: Use descriptive names like --primary-color, not --c1
  • Organization: Group related variables (colors, spacing, etc.)
  • Consistency: Use a consistent naming convention throughout
  • Documentation: Comment complex variable definitions
  • Scoping: Use global variables sparingly, prefer local scope when possible

Browser Support

CSS variables (custom properties) are supported in all modern browsers. Internet Explorer 11 and older versions do not support them.
