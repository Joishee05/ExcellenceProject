Forms

Lesson 9 of 15

Forms collect user input through various interactive elements. HTML forms use the <form> element to group input fields, buttons, and other controls. Forms are essential for user interaction: contact forms, logins, surveys, and more.

What Are Forms?

A form is a collection of input elements that gather information from users. Forms connect to backend servers to process data. Every form needs a submit button to send collected data for processing.

Common Form Uses:

  • Contact forms and feedback
  • Login and registration
  • Search functionality
  • Surveys and questionnaires
  • Checkout and payment

Basic Form Structure

<form action="/submit" method="POST">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <button type="submit">Submit</button>
</form>

Form Attributes:

  • action: URL where form data is sent
  • method: "GET" (visible in URL) or "POST" (hidden, secure)
  • name: Identifies form for JavaScript

Input Types

type="text"

Single line text input

type="email"

Email input with validation

type="password"

Hides typed characters

type="checkbox"

Multiple selections allowed

type="radio"

Only one selection allowed

type="date"

Date picker input

Labels and Accessibility

Always use <label> tags linked to inputs via for and id attributes. Labels improve accessibility and make forms easier to use:

Proper Label Usage:

<label for="email">Email Address:</label>
<input type="email" id="email" name="email" required>

<label for="subscribe">
  <input type="checkbox" id="subscribe" name="subscribe">
  Subscribe to newsletter
</label>

Form Validation

Use HTML5 attributes to validate input before submission:

Validation Attributes:

  • required: Field must be filled
  • minlength/maxlength: Control text length
  • min/max: Control number range
  • pattern: Match regex pattern

Validation Example:

<input type="text" name="username" minlength="3" maxlength="20" required>
<input type="number" name="age" min="18" max="120">
<input type="email" name="email" required>

Textarea and Select

For longer text and predefined options, use <textarea> and <select>:

Textarea for Long Text:

<label for="message">Message:</label>
<textarea id="message" name="message" rows="5" cols="40"></textarea>

Select Dropdown:

<label for="country">Country:</label>
<select id="country" name="country">
  <option value="">Select a country</option>
  <option value="usa">United States</option>
  <option value="uk">United Kingdom</option>
</select>

Form Best Practices

✓ DO
  • ✓ Always use labels
  • ✓ Mark required fields
  • ✓ Use appropriate input types
  • ✓ Validate input
  • ✓ Use POST for sensitive data
✗ DON'T
  • ✗ Skip labels
  • ✗ Use GET for passwords
  • ✗ Make forms overly complex
  • ✗ Reset forms unnecessarily
  • ✗ Disable autocomplete for passwords

Complete Form Example

<form action="/contact" method="POST">
  <h2>Contact Us</h2>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="message">Message:</label>
  <textarea id="message" name="message" rows="5" required></textarea>

  <button type="submit">Send</button>
</form>

Why Forms Matter

Understanding forms is crucial because: Forms are how websites collect user input and communicate with users. Every interactive website needs forms for user interaction, feedback, and data collection. Proper form design ensures usability, accessibility, and security.
Key Takeaways
  • Forms collect user input with <form> element
  • Use action attribute to specify where data is sent
  • Use method="POST" for secure data, method="GET" for non-sensitive
  • Always use <label> tags for accessibility
  • Input types: text, email, password, checkbox, radio, date, etc.
  • Use validation attributes: required, minlength, min, max, pattern
  • Use <textarea> for long text input
  • Use <select> for predefined options
