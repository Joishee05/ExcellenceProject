Comments in HTML

Lesson 12 of 15

HTML comments let you add notes and explanations in your code that don't display on the webpage. Comments help you document your work, explain complex sections, and leave reminders for future edits. They're invisible to visitors but visible to developers.

What Are HTML Comments?

Comments are text you include in your HTML file to explain what the code does. They're not rendered by browsers and won't appear on the webpage. Comments are purely for developers reading and maintaining the code.

Comment Syntax:

<!-- This is a comment -->

The structure is: <!-- starts the comment, --> ends it, and everything between is the comment text.

Single-Line Comments

Use single-line comments for brief explanations:

<!-- This is the header section -->
<header>
  <h1>Welcome</h1>
</header>

Real-World Single-Line Example:

<!-- Navigation menu -->
<nav>
  <a href="index.html">Home</a>
  <a href="about.html">About</a>
</nav>

Multi-Line Comments

For longer explanations, use multi-line comments:

<!--
  This section contains the main content
  of the webpage. It includes featured products
  and customer testimonials.
-->

Best Format for Multi-Line Comments:

<!--
  USER REGISTRATION FORM
  Allows new users to create accounts
  Fields: name, email, password
  Last updated: 2024
-->

Common Comment Uses

  • Explaining Sections – Mark where major sections begin/end
  • Temporary Disabling – Comment out code you might need later
  • TODO Notes – Remind yourself of unfinished work
  • Explaining Complex Code – Clarify non-obvious HTML structures
  • Documenting Changes – Note who changed what and when
  • Browser Compatibility – Explain workarounds

Comment Examples

Example 1: Section Markers

<!-- ===== HEADER SECTION ===== -->
<header>
  <h1>My Website</h1>
</header>

<!-- ===== MAIN CONTENT ===== -->
<main>
  <p>Welcome!</p>
</main>

<!-- ===== FOOTER ===== -->
<footer>
  <p>&copy; 2024</p>
</footer>

Example 2: TODO Notes

<!-- TODO: Add mobile responsiveness -->
<div class="product-gallery">
  <!-- TODO: Optimize image loading -->
  <img src="product1.jpg" alt="Product">
</div>

Example 3: Temporarily Disabling Code

<!--
<form action="old-script.php">
  <input type="text" placeholder="Old form">
</form>
-->

<form action="new-script.php">
  <input type="text" placeholder="New form">
</form>

Example 4: Explaining Complex Markup

<!--
  This nested structure creates a responsive grid
  The flex container adapts to mobile devices
-->
<div class="grid-container">
  <div class="grid-item">Item 1</div>
</div>

Comment Best Practices

✓ DO
  • ✓ Keep comments clear and concise
  • ✓ Update comments when code changes
  • ✓ Use comments to explain WHY, not what
  • ✓ Mark major sections clearly
  • ✓ Comment before complex code
✗ DON'T
  • ✗ Write obvious comments
  • ✗ Leave outdated comments
  • ✗ Over-comment everything
  • ✗ Use comments instead of refactoring
  • ✗ Nest comments inside each other

Comment Guidelines

❌ Bad Comment (Obvious):

<!-- Create a paragraph -->
<p>This is content</p>

✓ Good Comment (Explanatory):

<!-- Paragraph explains key features to first-time visitors -->
<p>This is content</p>

Important Comment Rules

✓ Correct: <!-- Comment -->

✗ Wrong: <! -- Comment -- >

✓ Correct: <!-- Two hyphens to close -->

✗ Wrong: <!--- Three hyphens ->

Comments Don't Display in Browser

Even if you add a comment, it won't appear on the webpage. Right-click "View Page Source" to see comments in the HTML code, but normal visitors can't see them.

Why Comments Matter

Comments are crucial for: Documenting your code for future edits, helping team members understand your work, explaining non-obvious design decisions, temporarily disabling code for testing, and creating a maintainable codebase. Comments save time when returning to code weeks later.
Key Takeaways
  • Comments use syntax: <!-- comment text -->
  • Comments are invisible in browsers
  • Use comments to explain WHY, not WHAT
  • Mark sections with clear comment headers
  • Temporarily disable code by commenting it out
  • Keep comments concise and up-to-date
  • View comments using "View Page Source"
  • Multi-line comments span across lines
