Variables & Data Types

Lesson 2 of 15

Variables are containers for storing data values. In JavaScript, you can declare variables using let, const, or var.

Declaring Variables

Using let

The let keyword declares a block-scoped variable that can be reassigned:

let name = "John";
let age = 25;
let isStudent = true;

// You can reassign values
age = 26;
name = "Jane";

Using const

The const keyword declares a block-scoped variable that cannot be reassigned:

const PI = 3.14159;
const SCHOOL = "Saint Louis University";

// This would cause an error:
// PI = 3.14; // Error!

Using var

The var keyword is the old way of declaring variables (avoid using it in modern JavaScript):

var oldStyle = "This is old";
// var has function scope, not block scope

Best Practice: Use const by default. Only use let when you need to reassign the variable. Avoid var.

Variable Naming Rules

  • Variable names can contain letters, digits, underscores, and dollar signs
  • Variable names must begin with a letter, $, or _
  • Variable names are case sensitive (myVar and myvar are different)
  • Reserved words (like JavaScript keywords) cannot be used as variable names
// Valid variable names
let firstName = "John";
let _privateVar = "secret";
let $price = 19.99;
let user123 = "user";

// Invalid variable names
// let 123user = "invalid";  // Can't start with number
// let first-name = "John";  // Can't use hyphens
// let let = "invalid";      // Can't use keywords

Variable Scope

Block Scope: Variables declared with let and const inside a block { } are only accessible within that block:

{
    let blockVar = "I'm inside a block";
    console.log(blockVar); // Works!
}
// console.log(blockVar); // Error! Not accessible here

Global Scope: Variables declared outside any function or block are global:

let globalVar = "I'm global";

function myFunction() {
    console.log(globalVar); // Works! Can access global var
}

myFunction();
Key Concept: Variable Declaration

Variables are named containers that hold values. JavaScript remembers the value until you change it.

  • const - Use by default (protects you from accidental changes)
  • let - For values that MUST change (recommended over var)
  • var - Old way (avoid in modern JavaScript)

Rule: Use const first, switch to let only if needed

Variable Declaration Comparison

Keyword Can Reassign? When to Use Example
const No Values that won't change const PI = 3.14;
let Yes Values that will change let counter = 0;
var Yes Avoid (confusing scope) var old = 'outdated';

Real-World Examples

Example 1: User Profile

// Create a user profile with constants and variables const userId = 12345; // Never changes const createdDate = '2025-01-01'; // Fixed date let username = 'john_doe'; // Can be updated let email = 'john@example.com'; // Can be updated let profileViews = 150; // Changes frequently console.log(`User ${username} has ${profileViews} profile views`);

Example 2: Calculating Costs

const TAX_RATE = 0.08; // Fixed tax rate const SHIPPING = 5.99; // Fixed shipping cost let itemPrice = 99.99; // Item cost let quantity = 3; // Can change let subtotal = itemPrice * quantity; let tax = subtotal * TAX_RATE; let total = subtotal + tax + SHIPPING; console.log('Subtotal:', subtotal); console.log('Tax:', tax); console.log('Total:', total);

Understanding Data Type Flow

How JavaScript determines data types: 
Variable Creation:
  ↓
Assign Value:
  let x = "hello"  →  JavaScript sees quotes  →  Type: String
  let y = 42       →  JavaScript sees number  →  Type: Number
  let z = true     →  JavaScript sees boolean →  Type: Boolean
  ↓
Use typeof operator to check:
  console.log(typeof x);  // "string"
  console.log(typeof y);  // "number"
  console.log(typeof z);  // "boolean"

Key Takeaways

  • Variables are containers that store data values
  • Use const by default for values that won't change
  • Use let only when you need to reassign a variable
  • Avoid var - it has confusing scope rules
  • Every value in JavaScript has a data type (string, number, boolean, etc.)
  • Use descriptive variable names to make code readable
  • The typeof operator tells you a value's type
