Lesson 11: Media Queries

Lesson 11 of 15

What You'll Learn

In this lesson, you will master media queriesthe CSS feature that enables responsive design. You will learn to target different screen sizes, device types, and user preferences.

What are Media Queries?

Media queries are CSS rules that apply styles based on device characteristics. They are the foundation of responsive design.

@media (condition) { /* Styles applied when condition is true */ }

Basic Media Query Syntax

Breakpoint-Based Media Query

/* All screens 768px and wider */ @media (min-width: 768px) { .container { width: 750px; } } /* Screens 767px and narrower */ @media (max-width: 767px) { .container { width: 100%; } }

What This Does:

  • @media (min-width: 768px) { } - "Only apply these styles if screen is at least 768px wide". Like a conditional in programming
  • Inside the braces { }: CSS that only applies if condition is true
  • width: 750px; - On tablets and desktops (768px+), make container 750px wide
  • @media (max-width: 767px) { } - "Only apply if screen is 767px or narrower". For phones
  • width: 100%; - On phones, make container full width (100% of available space)

Range Media Query

/* Between 768px and 1024px */ @media (min-width: 768px) and (max-width: 1024px) { .container { width: 720px; } }

Common Media Query Features

Width and Height

@media (min-width: 600px) { } /* Viewport at least 600px wide */ @media (max-width: 599px) { } /* Viewport at most 599px wide */ @media (min-height: 400px) { } /* Viewport at least 400px tall */ @media (max-height: 399px) { } /* Viewport at most 399px tall */

What These Do:

  • (min-width: 600px): "Is the screen at least 600 pixels wide?" If YES, apply these styles
  • (max-width: 599px): "Is the screen 599 pixels or narrower?" If YES, apply these styles
  • Typical breakpoints: 480px (phones), 768px (tablets), 1024px (desktops)
  • (min-height): Check if viewport is tall enough. (max-height): Check if it's not too tall
  • Why useful: Different designs for different screen sizes. Phones need narrow layouts, desktops can use 3 columns

Device Type

@media screen { } /* Any screen device */ @media print { } /* Printing/print preview */ @media (display-mode: fullscreen) { } /* Fullscreen mode */

Orientation

@media (orientation: portrait) { /* Viewport height greater than width */ } @media (orientation: landscape) { /* Viewport width greater than height */ }

Advanced Media Query Features

Color and Color Gamut

@media (color) { /* Color-capable device */ } @media (color-gamut: srgb) { /* Standard color range */ } @media (color-gamut: p3) { /* Wide color range */ }

Hover and Pointer

/* Device supports hover (desktop with mouse) */ @media (hover: hover) { button:hover { background: #10b981; } } /* Device doesn't support hover (touchscreen) */ @media (hover: none) { button { background: #10b981; /* Show active state by default */ } } /* Pointer type */ @media (pointer: fine) { /* Mouse or stylus - precise pointer */ } @media (pointer: coarse) { /* Touchscreen - larger touch targets */ }

Prefers Reduced Motion

Respect user accessibility preferences for reduced animations.

/* Default animation */ .animated-box { animation: spin 2s infinite; } /* Respect user preference */ @media (prefers-reduced-motion: reduce) { .animated-box { animation: none; } }

Dark Mode Media Query

Detect if user prefers dark theme.

/* Light mode (default) */ body { background: white; color: #333; } /* Dark mode */ @media (prefers-color-scheme: dark) { body { background: #1a1a1a; color: #f0f0f0; } a { color: #64b5f6; } }

Logical Operators in Media Queries

AND Operator

/* Both conditions must be true */ @media (min-width: 768px) and (orientation: landscape) { /* Large screen AND landscape */ }

OR Operator (Comma)

/* Any condition can be true */ @media (max-width: 600px), (orientation: portrait) { /* Mobile OR portrait orientation */ }

NOT Operator

@media not screen { /* Not a screen (e.g., print) */ }

Common Responsive Patterns

Mobile-First CSS

/* Mobile first (no media query) */ .container { display: block; width: 100%; } /* Tablet and up */ @media (min-width: 768px) { .container { display: flex; width: 80%; margin: 0 auto; } } /* Desktop and up */ @media (min-width: 1024px) { .container { width: 1000px; } }

Responsive Columns

/* Mobile: 1 column */ .grid { display: grid; grid-template-columns: 1fr; } /* Tablet: 2 columns */ @media (min-width: 768px) { .grid { grid-template-columns: 1fr 1fr; } } /* Desktop: 3 columns */ @media (min-width: 1024px) { .grid { grid-template-columns: 1fr 1fr 1fr; } }

Conditional Navigation

/* Mobile: Hamburger menu */ .nav { display: none; /* Hidden by default */ } .hamburger { display: block; } /* Tablet and up: Horizontal menu */ @media (min-width: 768px) { .nav { display: flex; gap: 30px; } .hamburger { display: none; } }

Media Query Best Practices

  • Mobile-first: Start with mobile styles, enhance for larger screens
  • Logical breakpoints: Choose breakpoints where content needs to shift, not device sizes
  • Test thoroughly: Check all breakpoints and edge cases
  • Avoid too many breakpoints: 2-4 breakpoints usually sufficient
  • Use readable breakpoint values: Stick to common sizes (480px, 768px, 1024px, 1280px)

Browser Support

Media queries have excellent support in all modern browsers. Older versions of Internet Explorer 8 and below have no support.
