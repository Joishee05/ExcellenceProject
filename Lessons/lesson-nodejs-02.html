Setting Up Node.js

Lesson 22 of 23

Why Proper Setup Matters

Before you can start building Node.js applications, you need to properly install and configure Node.js and npm on your machine. This lesson walks you through the entire setup process, ensuring your development environment is ready for the next lesson where you'll create your first server.

What is npm?

npm (Node Package Manager) is the default package manager for Node.js. It comes automatically when you install Node.js. npm allows you to:

  • Install packages: Download code libraries written by other developers
  • Manage dependencies: Keep track of what packages your project needs
  • Version control: Ensure consistent package versions across your team
  • Share code: Publish your own packages for others to use

Think of npm as an app store for code. Instead of writing everything from scratch, you can download and use thousands of pre-built modules to extend your Node.js applications.

Installation Steps

Node.js is available for Windows, macOS, and Linux. We recommend installing the LTS (Long Term Support) version, which is the most stable and widely supported version. Follow the instructions for your operating system:

Installation on Windows

  1. Visit https://nodejs.org in your web browser
  2. You'll see two versions: LTS and Current. Click the LTS button to download
  3. The installer (.msi file) will download. Open it and follow the setup wizard
  4. Accept the license agreement and choose the default installation path
  5. Keep "npm package manager" checked (it should be by default)
  6. Complete the installation
  7. Restart your computer to ensure the PATH is updated

Installation on macOS

You have two options on macOS:

# Option 1: Using Homebrew (recommended) brew install node # Option 2: Download installer from nodejs.org # Visit https://nodejs.org and download the macOS installer

If you don't have Homebrew installed, visit https://brew.sh to install it first. Homebrew is a package manager for macOS that makes installing tools much easier.

Installation on Linux

Use your distribution's package manager:

# Ubuntu / Debian sudo apt-get update sudo apt-get install nodejs npm # Fedora sudo dnf install nodejs npm # Arch sudo pacman -S nodejs npm

Verifying Your Installation

After installation, verify that Node.js and npm installed correctly by checking their versions. Open your terminal or command prompt and run:

node --version npm --version

You should see version numbers appear (for example: v18.16.0 and 9.6.7). If you see version numbers, congratulationsâ€”your installation was successful!

Understanding Your Node.js Installation

When you install Node.js, you're installing two main components:

  1. Node.js Runtime: The JavaScript engine that executes your code server-side
  2. npm: The package manager for downloading and managing dependencies

These two components work together. Node.js runs your JavaScript code, and npm helps you manage the libraries your code depends on.

Creating Your First Project Directory

Now that Node.js is installed, let's create your first Node.js project directory:

# Create a new folder for your project mkdir my-first-node-app cd my-first-node-app # Initialize a Node.js project (this creates package.json) npm init -y

The npm init -y command creates a package.json file in your directory. This file tracks your project's metadata and dependencies.

Understanding package.json

The package.json file is the heart of every Node.js project. Here's what a basic one looks like:

{ "name": "my-first-node-app", "version": "1.0.0", "description": "", "main": "index.js", "scripts": { "test": "echo \"Error: no test specified\" && exit 1" }, "keywords": [], "author": "", "license": "ISC" }

Key fields:

  • name: Your project's name
  • version: Current version of your project
  • main: The entry point file (usually index.js)
  • scripts: Commands you can run with npm run
  • dependencies: Packages your project needs (added when you install packages)

Installing Packages with npm

One of npm's most powerful features is the ability to install packages. For example, if you want to use the popular Express.js web framework:

npm install express

This command:

  1. Downloads the Express package from npm's online registry
  2. Saves it in a folder called node_modules
  3. Updates your package.json to record the dependency

Later, when you share your project, others can run npm install and automatically get the same packages.

Common Setup Issues

Issue 1: "node command not found"

  • Problem: Node.js isn't in your system PATH
  • Solution: Restart your computer after installation, or reinstall Node.js

Issue 2: npm conflicts with existing packages

  • Problem: You're trying to install a package that conflicts with an existing one
  • Solution: Delete the node_modules folder and run npm install again

Issue 3: Permission errors on macOS/Linux

  • Problem: npm install fails with permission denied errors
  • Solution: Use nvm (Node Version Manager) to manage Node.js instead of sudo

Using Node.js in Your Projects

After setup, you can create Node.js files (typically with .js extension) and run them:

# Create a simple JavaScript file echo "console.log('Hello from Node.js!');" > app.js # Run it with Node node app.js

The output should be: Hello from Node.js!

Best Practices for Node.js Setup

  • Always use LTS versions in production environments for maximum stability
  • Keep Node.js updated but test updates before deploying to production
  • Use .gitignore: Always exclude node_modules from version control. Create a .gitignore file with node_modules on a line
  • Document your setup: Include setup instructions in your project's README
  • Use project-specific versions: Consider using nvm for managing multiple Node.js versions
  • Share package.json only: When sharing projects, include package.json so others can install dependencies

What's Next?

Key Takeaways

  • Node.js is JavaScript for servers; npm is Node.js's package manager
  • Installation is straightforward: download from nodejs.org or use package managers (Homebrew, apt-get, etc.)
  • Always verify installation by checking node --version and npm --version
  • package.json is your project's configuration file and tracks dependencies
  • npm makes it easy to install pre-built packages instead of writing everything yourself
  • Proper setup and project initialization is crucial for maintainable Node.js applications
