Lesson 4: Box Model

Lesson 4 of 15

What You'll Learn

In this fundamental lesson, you will master the CSS box modelthe core concept for understanding layout in CSS. You will learn how elements are sized and spaced using margin, border, padding, and content.

What is the Box Model?

Every HTML element is surrounded by a box. The CSS box model describes this box, which consists of four layers from inside out: content, padding, border, and margin. Understanding the box model is essential for controlling spacing and layout.

The Four Layers of the Box Model

1. Content

The innermost layer where the actual content (text, images, etc.) appears. Its size is controlled by the width and height properties.

2. Padding

The space inside the element, between the content and the border. Padding is inside the element and uses the element's background color.

padding: 20px; /* All sides */ padding: 10px 20px; /* Top/Bottom 10px, Left/Right 20px */ padding: 10px 15px 10px 20px; /* Top, Right, Bottom, Left */ padding-top: 10px; padding-right: 15px; padding-bottom: 10px; padding-left: 20px;

What Padding Does:

  • padding: 20px; - Adds 20 pixels of space on all 4 sides (top, right, bottom, left)
  • padding: 10px 20px; - Shorthand. Top and bottom get 10px, left and right get 20px
  • padding: 10px 15px 10px 20px; - Four values: top, right, bottom, left (clockwise order)
  • padding-top: 10px; - Target just one side. Useful when you need different values
  • Inside vs Outside: Padding is INSIDE the box so the background color fills the padding area too

3. Border

The border goes around the padding and creates a visible outline around the element.

border: 1px solid #ddd; /* Width, Style, Color */ border: 2px dashed red; /* Dashed border */ border: 3px double black; /* Double border */ border-top: 1px solid #333; /* Top border only */ border-radius: 8px; /* Rounded corners */

What Border Does:

  • border: 1px solid #ddd; - Three parts: 1px (width), solid (style), #ddd (light gray color). Creates a thin solid line
  • 1px: The thickness of the border in pixels
  • solid: Style of line. Can be: solid, dashed, dotted, double, groove, ridge, etc.
  • border: 2px dashed red; - Creates a dashed red border 2 pixels thick
  • border-top: 1px solid #333; - Target just one side for selective borders
  • border-radius: 8px; - Rounds the corners. Bigger number = rounder corners

4. Margin

The space outside the element, between its border and adjacent elements. Margin is outside the element and is transparent (doesn't inherit background color).

margin: 20px; /* All sides */ margin: 10px 20px; /* Top/Bottom 10px, Left/Right 20px */ margin: 10px 15px 10px 20px; /* Top, Right, Bottom, Left */ margin-top: 10px; margin-right: 15px; margin-bottom: 10px; margin-left: 20px;

What Margin Does:

  • margin: 20px; - Adds 20 pixels of space on all 4 sides (OUTSIDE the element)
  • Padding vs Margin: Padding = inside the box (colors the background). Margin = outside (space between elements)
  • margin: 10px 20px; - Top and bottom get 10px, left and right get 20px
  • margin: 10px 15px 10px 20px; - Top, right, bottom, left (clockwise starting from top)
  • margin-top: 10px; - Target just one side. Useful for fine control
  • Why it matters: Margin controls how close elements get to each other. Large margin = lots of space

Box-Sizing Property

By default, the width and height properties only apply to the content area, not including padding and border. The box-sizing property changes this behavior.

Content-Box (Default)

/* Default behavior */ box-sizing: content-box; /* A box with width: 200px and padding: 20px */ /* Total width on page = 200px (content) + 40px (padding) = 240px */

Border-Box (Recommended)

/* Include padding and border in width */ box-sizing: border-box; /* A box with width: 200px and padding: 20px */ /* Total width on page = 200px (includes padding and border) */

What This Does:

  • box-sizing: border-box; - Changes how width is calculated. Makes sizing more predictable
  • With border-box: width: 200px = 200px total INCLUDING padding and border
  • Why easier: You say "I want 200px width" and it stays 200px no matter the padding. No math needed!
  • content-box (default): width: 200px + padding: 20px = 240px actual size. Confusing for beginners!
  • Best practice: Put "* { box-sizing: border-box; }" at the top of your CSS. Every element will behave intuitively

Margin Collapse

An important concept: when two block elements have margins that touch, only the larger margin is used, not both added together. This is called margin collapse.

/* Two paragraphs */ p { margin-bottom: 20px; margin-top: 20px; } /* Between two p elements, spacing is 20px (not 40px) because margins collapse. Only the larger margin is used. */

Practical Box Model Examples

Creating a Card Component

.card { background: white; border: 1px solid #ddd; border-radius: 8px; padding: 20px; /* Space inside card */ margin: 15px; /* Space around card */ box-shadow: 0 2px 8px rgba(0,0,0,0.1); } .card h2 { margin-top: 0; /* Remove top margin from heading */ margin-bottom: 10px; } .card p { margin: 0; /* Reset default paragraph margins */ }

Centering Content with Margin

.container { width: 1200px; margin: 0 auto; /* Center horizontally: 0 top/bottom, auto left/right */ padding: 20px; /* Space inside container */ }

Box Model Best Practices

  • Use border-box: Apply box-sizing: border-box globally for easier sizing
  • Reset Defaults: Remove unwanted default margins from elements like p, h1-h6, ul, ol
  • Be Consistent: Use consistent spacing values throughout your design
  • Understand Margin Collapse: Remember that adjacent margins collapse
  • Use Padding for Inside Space: Use padding to create space within an element
  • Use Margin for Outside Space: Use margin to create space between elements

Debugging Box Model Issues

In your browser's developer tools (F12), use the Inspect Element feature. The Elements panel shows the box model diagram for any element, letting you see exact padding, border, and margin values.
