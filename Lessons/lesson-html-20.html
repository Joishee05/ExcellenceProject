Understanding HTTP Requests

HTMX lets you make different types of HTTP requests directly from HTML attributes. These are the same types of requests your browser makes when you click a link or submit a form, but now you can do it with HTMX for better control.

GET Requests

Use hx-get to fetch data from the server without sending any data along with the request:

<button hx-get="/api/greeting">Get Greeting</button>

This is useful for:

  • Loading content when a button is clicked
  • Fetching data without modifying anything on the server
  • Simple page loads without reloading the entire page

POST Requests

Use hx-post to send data to the server and create or update something:

<form hx-post="/api/users"> <input type="text" name="username" placeholder="Username"> <button>Create User</button> </form>

POST requests are useful for:

  • Creating new items (users, posts, todos, etc.)
  • Submitting forms without page reload
  • Sending data that creates a change on the server

PUT Requests

Use hx-put to update existing data on the server:

<button hx-put="/api/users/123" hx-target="#user-info"> Update User </button>

PUT requests are useful for:

  • Updating existing records (editing user profile, changing settings)
  • Replacing entire resources with new data
  • Modifying items that already exist

DELETE Requests

Use hx-delete to remove data from the server:

<button hx-delete="/api/users/123" hx-confirm="Delete this user?"> Delete User </button>

DELETE requests are useful for:

  • Removing items from a list
  • Deleting records from the database
  • Cleanup operations

PATCH Requests

Use hx-patch to partially update data (just the fields that changed):

<input type="text" name="email" placeholder="New email"> <button hx-patch="/api/users/123">Update Email Only</button>

PATCH requests are useful for:

  • Updating just one or two fields
  • More efficient than PUT for partial updates
  • Reducing the amount of data sent to the server

Request Attributes Comparison

Attribute HTTP Method Use Case Example
hx-get GET Fetch data without changes Load comments, get content
hx-post POST Create new data Submit form, add item
hx-put PUT Replace entire resource Update user profile
hx-patch PATCH Update specific fields Change user status only
hx-delete DELETE Remove data Delete comment, remove item
