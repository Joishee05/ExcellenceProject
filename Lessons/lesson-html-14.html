Audio & Video in HTML

Lesson 14 of 15

HTML5 provides native support for embedding audio and video files directly in webpages. The <audio> and <video> tags eliminate the need for plugins and provide built-in player controls. This makes media accessible and user-friendly.

HTML Audio

The <audio> tag embeds audio files and provides controls for playing, pausing, and adjusting volume.

Basic Audio Syntax:

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  Your browser doesn't support audio element.
</audio>

Audio Attributes:

controls – Show play/pause controls

autoplay – Auto start playback

loop – Repeat audio

muted – Start muted

preload – Load audio on page load

volume – Set initial volume

Audio Example with Multiple Formats:

<audio controls autoplay muted>
  <source src="music.mp3" type="audio/mpeg">
  <source src="music.ogg" type="audio/ogg">
  Audio not supported in this browser.
</audio>

HTML Video

The <video> tag embeds video files with built-in player controls, autoplay, and responsive sizing.

Basic Video Syntax:

<video controls width="400" height="300">
  <source src="video.mp4" type="video/mp4">
  Your browser doesn't support the video element.
</video>

Video Attributes:

width – Video width in pixels

height – Video height in pixels

controls – Show player controls

autoplay – Auto start playback

loop – Repeat video

muted – Start muted

poster – Image before play

preload – Load video on page load

Video Example with Poster:

<video controls width="640" height="360" poster="thumbnail.jpg">
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  Video not supported in this browser.
</video>

Supported File Formats

Audio Formats:

  • MP3 (.mp3) – Most compatible, compressed audio
  • OGG (.ogg) – Open format, good compression
  • WAV (.wav) – Uncompressed, larger file size
  • M4A (.m4a) – Apple format, good for iTunes

Video Formats:

  • MP4 (.mp4) – Most compatible, H.264 codec
  • WebM (.webm) – Modern open format, VP8 codec
  • Ogg Theora (.ogv) – Older open format

Source Tag and Browser Compatibility

The <source> tag specifies multiple file formats. Browsers use the first compatible format:

<video controls>
  <!-- Browser uses first supported format -->
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <p>Fallback: <a href="video.mp4">Download video</a></p>
</video>

Responsive Video

Make videos responsive with CSS:

/* CSS */
video {
  max-width: 100%;
  height: auto;
}

<!-- HTML -->
<video controls>
  <source src="video.mp4" type="video/mp4">
</video>

Best Practices

✓ DO
  • ✓ Use multiple formats for compatibility
  • ✓ Always include controls attribute
  • ✓ Add fallback text/link
  • ✓ Compress media files
  • ✓ Use poster image for videos
✗ DON'T
  • ✗ Use large uncompressed files
  • ✗ Autoplay without user interaction
  • ✗ Forget alt text/captions
  • ✗ Use outdated embedded players
  • ✗ Skip fallback messages

Complete Audio & Video Example

<!-- Audio Player -->
<h3>Podcast</h3>
<audio controls preload="none">
  <source src="podcast.mp3" type="audio/mpeg">
  <source src="podcast.ogg" type="audio/ogg">
  Your browser doesn't support audio.
</audio>

<!-- Video Player -->
<h3>Tutorial Video</h3>
<video controls width="640" height="360" poster="thumbnail.jpg">
  <source src="tutorial.mp4" type="video/mp4">
  <source src="tutorial.webm" type="video/webm">
  <p><a href="tutorial.mp4">Download video</a></p>
</video>

Why Native Media is Better

Using HTML5 audio/video instead of Flash: Better performance, built-in controls, mobile-friendly, no plugin required, accessible to all devices, faster loading, better security, easier to customize with CSS and JavaScript.
Key Takeaways
  • <audio controls> embeds audio with controls
  • <video controls width="X" height="Y"> embeds video
  • Use <source> tag for multiple formats
  • Always include fallback text for unsupported browsers
  • MP4 video and MP3 audio work on all modern browsers
  • Add poster attribute to show thumbnail before play
  • Use CSS to make video responsive
  • Always compress media to improve page speed
