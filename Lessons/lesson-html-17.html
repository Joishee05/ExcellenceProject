Core HTMX Attributes

Lesson 17 of 18

HTMX provides a powerful set of attributes that you add to HTML elements to create dynamic interactions. These attributes tell HTMX what to do, when to do it, and where to put the response.

The Essential Attributes

Request Attributes

These attributes specify what HTTP request to make:

Attribute Purpose Example
hx-get Make a GET request to the specified URL hx-get="/data"
hx-post Make a POST request to the specified URL hx-post="/submit"
hx-put Make a PUT request to the specified URL hx-put="/update/1"
hx-delete Make a DELETE request to the specified URL hx-delete="/remove/1"
hx-patch Make a PATCH request to the specified URL hx-patch="/modify/1"

Targeting Attributes

These attributes specify where to insert the response:

Attribute Purpose Example
hx-target Specify which element receives the response (CSS selector) hx-target="#results"
hx-select Select a specific part of the response to use hx-select=".user-card"

Swap Attributes

These attributes specify how to insert the response:

Attribute Purpose
hx-swap="innerHTML" Replace the inner HTML (default behavior)
hx-swap="outerHTML" Replace the entire element
hx-swap="beforebegin" Insert before the element
hx-swap="afterbegin" Insert as first child of element
hx-swap="beforeend" Insert as last child of element
hx-swap="afterend" Insert after the element

Trigger Attributes

These attributes specify when to make the request:

Attribute Purpose Example
hx-trigger Specify which event triggers the request (default: click for buttons, change for inputs) hx-trigger="keyup"

Practical Examples

Example 1: Search Results

<input type="text" placeholder="Search users..." hx-get="/search" hx-trigger="keyup" hx-target="#results" /> <div id="results"></div>

This searches as the user types, updates results in real-time without a submit button!

Example 2: Delete with Confirmation

<button hx-delete="/user/123" hx-target="closest .user-card" hx-swap="outerHTML" onclick="return confirm('Are you sure?')"> Delete User </button>

This deletes a user and removes the entire card from the page.

Example 3: Load More Button

<button hx-get="/more-posts?page=2" hx-target="#posts" hx-swap="beforeend"> Load More Posts </button> <div id="posts">...existing posts...</div>

This loads additional posts and appends them to the existing list.

Common Patterns

Form Submission

<form hx-post="/login" hx-target="#auth"> <input type="email" name="email" required/> <input type="password" name="password" required/> <button type="submit">Login</button> </form> <div id="auth"></div>

HTMX automatically captures form data and sends it as the request body.

Conditional Requests

<button hx-get="/data" hx-trigger="click" hx-target="#content"> Refresh Data </button>
Key Takeaways
  • hx-get/post/put/delete - Specify the HTTP request method and URL
  • hx-target - Specify where to put the response
  • hx-swap - Specify how to insert the response (innerHTML, outerHTML, etc.)
  • hx-trigger - Specify when to make the request (click, keyup, change, etc.)
  • These attributes work together to create powerful dynamic interactions
