What is Node.js?

Lesson 21 of 23

Introduction to Node.js

Node.js is a JavaScript runtime built on Chrome's V8 engine. It allows you to run JavaScript on the server side, enabling full-stack JavaScript development. Instead of JavaScript only running in web browsers, you can now use JavaScript to build web servers, APIs, and backend systems.

The Problem Node.js Solves

Traditionally, developers had to learn multiple languages:

  • Frontend (Browser): JavaScript
  • Backend (Server): Python, Java, PHP, Ruby, etc.

With Node.js, you can use JavaScript for both frontend AND backend, making development faster and allowing developers to specialize in one language.

Server-Side JavaScript

Before Node.js (pre-2009), JavaScript only ran in web browsers. This was a limitation because:

  • JavaScript couldn't access the file system
  • JavaScript couldn't create web servers
  • JavaScript couldn't interact with databases directly
  • You needed a different language for backend development

Node.js changed this by bringing JavaScript to the server, enabling it to do everything traditional backend languages can do.

Key Features of Node.js

Event-Driven Architecture: Node.js uses an event-driven model where the server responds to events (like incoming requests) rather than blocking and waiting.
Non-Blocking I/O: When Node.js needs to read a file or query a database, it doesn't pause the entire server. Instead, it handles other requests while waiting for the operation to complete. This makes Node.js very efficient.
Fast Execution: Node.js uses the V8 engine (the same engine Google Chrome uses), which is incredibly fast at executing JavaScript.
Large Package Ecosystem (npm): npm (Node Package Manager) has millions of packages available. Need a CSV parser? Date library? HTTP client? There's likely an npm package for it.
Scalability: Node.js is great for building scalable applications that can handle many concurrent connections efficiently.

Node.js vs Browser JavaScript

Feature Browser JavaScript Node.js
File System Access No (security) Yes
DOM Manipulation Yes No
Database Access No (security) Yes
Create Web Server No Yes
Access System Environment Limited Full Access

Real-World Examples Using Node.js

  • Web Servers: Express.js is the most popular web framework for Node.js
  • REST APIs: Node.js powers countless APIs (including DigitalBonesBox!)
  • Real-Time Applications: Socket.io enables real-time communication
  • Command-Line Tools: Many popular CLI tools are built with Node.js
  • Microservices: Node.js is lightweight and perfect for microservice architectures

The JavaScript Everywhere Vision

Node.js enables "JavaScript everywhere" - using the same language across your entire application stack:

  • Frontend: React, Vue, Angular (all JavaScript)
  • Backend: Node.js with Express
  • Database: MongoDB (stores JSON, works great with JavaScript)
  • DevOps: Build scripts, deployment scripts (JavaScript)

Key Takeaways

Node.js is a JavaScript runtime that enables server-side development. It uses event-driven, non-blocking I/O for efficiency. With Node.js, you can build complete full-stack applications using only JavaScript. Understanding Node.js opens many possibilities for full-stack development and is essential knowledge for modern web developers.
