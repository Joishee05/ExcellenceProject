HTMX Events & Triggers

Lesson 22 of 23

What is hx-trigger?

The hx-trigger attribute specifies WHEN an HTMX request should be made. By default, most elements trigger on click, but you can change this to any JavaScript event.

Common Trigger Events

Here are the most common events you'll use:

  • click - When user clicks (default for buttons)
  • change - When form input value changes
  • keyup - When user types in an input
  • load - When page loads
  • submit - When form is submitted
  • focus - When element receives focus
  • blur - When element loses focus

Examples by Trigger Type

Click Trigger (Default)

<button hx-get="/action">Click Me</button>

Triggered when user clicks the button.

Change Trigger

<select hx-get="/filter" hx-trigger="change" hx-target="#results"> <option>All</option> <option>Active</option> <option>Completed</option> </select>

Triggers request when user selects a different option.

Keyup Trigger

<input type="text" hx-get="/search" hx-trigger="keyup" hx-target="#results" placeholder="Search...">

Makes a request each time user types. Great for live search!

Load Trigger

<div hx-get="/initial-content" hx-trigger="load"></div>

Automatically loads content when the page loads.

Submit Trigger

<form hx-post="/submit" hx-trigger="submit"> <input type="text" name="message"> <button type="submit">Send</button> </form>

Submits form data via HTMX instead of traditional form submission.

Modifying Triggers

Debouncing

Delay the request by a set time (useful for search to avoid too many requests):

<input type="text" hx-get="/search" hx-trigger="keyup delay:500ms" hx-target="#results">

Waits 500ms after user stops typing before making request.

Throttling

Limit request frequency:

<input type="text" hx-get="/search" hx-trigger="keyup throttle:1s">

Maximum one request per second while typing.

Multiple Triggers

Use multiple events:

<input type="text" hx-get="/validate" hx-trigger="change, blur">

Triggers on both change and blur events.

Real-World Examples

Live Search

<input type="text" placeholder="Search users..." hx-get="/search-users" hx-trigger="keyup delay:300ms" hx-target="#search-results"> <div id="search-results"></div>

Auto-Save Form

<textarea hx-post="/auto-save" hx-trigger="change delay:2s" name="content"></textarea>

Filter on Change

<select hx-get="/filter-items" hx-trigger="change" hx-target="#item-list"> <option value="all">All Items</option> <option value="active">Active Only</option> </select> <div id="item-list"></div>

Summary

  • hx-trigger controls WHEN requests happen
  • Common triggers: click, change, keyup, load, submit
  • Use delay:500ms to wait before requesting
  • Use throttle:1s to limit request frequency
  • Multiple triggers separated by commas work together
  • Combine with other attributes for powerful interactions
