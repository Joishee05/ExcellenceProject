Lesson 1: What is CSS?

Lesson 1 of 15

What You'll Learn

In this comprehensive lesson, you will learn the fundamentals of CSS, including what it is, why it's essential for modern web development, how it works with HTML, and how to structure CSS code effectively. By the end of this lesson, you'll have a solid foundation to move forward with more advanced CSS topics.

What is CSS?

CSS stands for Cascading Style Sheets. It is a language used to describe how HTML elements should be displayed on screen, on paper, or in other media. CSS is used to style and layout web pages by controlling fonts, colors, sizes, spacing, positioning, animations, and visual effects.

CSS was first introduced in 1996 and has evolved significantly over time. Modern CSS (CSS3 and beyond) includes advanced features like Flexbox, Grid, animations, and custom properties that make creating complex, responsive layouts much simpler than before.

Why is CSS Important?

CSS is absolutely essential to web development for several key reasons:

  • Visual Design: CSS enables you to create beautiful, visually appealing websites that stand out and engage users.
  • User Experience: Proper styling improves readability, navigation, and overall user experience. Good CSS makes sites intuitive and pleasant to use.
  • Responsive Design: CSS allows websites to adapt to different screen sizes (mobile, tablet, desktop), ensuring your site works everywhere.
  • Performance: CSS is extremely efficient. A small CSS file can style an entire website, reducing the amount of code needed.
  • Maintainability: Separating CSS from HTML makes code easier to maintain, update, and scale as projects grow.
  • Brand Consistency: CSS makes it easy to apply consistent styling across your entire website.
  • Accessibility: Proper CSS helps create accessible websites that work for users with disabilities.

How CSS Works with HTML

HTML and CSS work together but serve different purposes:

  • HTML (HyperText Markup Language): Provides the structure and content of a web page. It defines what elements exist on the page (headings, paragraphs, images, etc.).
  • CSS (Cascading Style Sheets): Controls the presentation and styling. It defines how those elements look (colors, fonts, sizes, positions, animations).

Think of HTML as the skeleton of a house (the structure) and CSS as the paint, furniture, and decorations (the appearance).

CSS Syntax: Rules and Properties

CSS code is organized into rules. Each rule consists of a selector and a declaration block:

selector { property: value; another-property: another-value; }

What This Code Does:

  • selector { } - The curly braces create a "rule block" that holds all the style instructions for that selector
  • property: - This is the CSS property name (what you want to change, like color or size)
  • value; - This is what you want to change it to. The semicolon at the end tells CSS "this instruction is done"

Breaking Down CSS Syntax

  • Selector: Targets which HTML elements to style (e.g., h1, .classname, #idname)
  • Property: Specifies what aspect of the element to change (e.g., color, font-size, margin)
  • Value: Specifies how to change that property (e.g., blue, 24px, 20px)
  • Declaration Block: Contains all the property-value pairs enclosed in curly braces

Practical CSS Examples

/* Make all h1 headings blue and larger */ h1 { color: blue; font-size: 32px; margin: 20px 0; } /* Style paragraphs with gray text and line spacing */ p { color: #666; line-height: 1.6; margin-bottom: 15px; }

Let's Break This Down:

  • h1 { } - This targets all h1 headings on the page. Anything inside these braces will apply to every h1
  • color: blue; - Changes the text color to blue. The word "blue" is the value we're giving to the color property
  • font-size: 32px; - Makes the text 32 pixels tall. "px" means pixels (a small unit of measurement on screens)
  • margin: 20px 0; - Adds 20 pixels of space above and below each h1, but no space on the left and right
  • p { } - This targets all paragraphs. Each paragraph will get these styles
  • color: #666; - Changes text to a gray color. The #666 is a special code for colors (called hexadecimal)
  • line-height: 1.6; - Makes space between lines of text. 1.6 means 1.6 times the normal height, making text easier to read
  • margin-bottom: 15px; - Adds 15 pixels of space below each paragraph so paragraphs don't touch each other

Three Ways to Add CSS to HTML

1. External CSS (Recommended)

Link to a separate CSS file in your HTML head. This is the most professional and maintainable approach.

<!-- In your HTML file --> <head> <link rel="stylesheet" href="styles.css"> </head>

What This Does:

  • <link ... > - This HTML tag tells the browser to load a CSS file
  • rel="stylesheet" - Tells the browser "this is a stylesheet" (CSS file)
  • href="styles.css" - Tells the browser where to find the CSS file. In this case, it's a file named "styles.css" in the same folder

Then you create a separate file called "styles.css" with all your CSS rules inside it. This is the professional way because:

  • Your HTML stays clean and focused on structure
  • Your CSS stays organized in its own file
  • You can use the same CSS file for multiple HTML pages
  • Browsers remember (cache) the CSS file, so pages load faster

2. Internal CSS

Write CSS directly in the HTML file using style tags in the head section.

<head> <style> h1 { color: blue; font-size: 24px; } </style> </head>

What This Does:

  • <style> ... </style> - This HTML tag tells the browser "everything inside here is CSS code, apply these styles to this page"
  • All your CSS rules go between the opening and closing style tags
  • This only affects the current HTML page, not other pages

When to Use This: Good for quick testing or small single-page projects. Not recommended for large websites.

3. Inline CSS

Apply CSS directly to individual HTML elements using the style attribute.

<h1 style="color: blue; font-size: 24px;">Hello World</h1>

What This Does:

  • style=" " - This is an HTML attribute (an extra piece of information on a tag) that holds CSS code
  • color: blue; font-size: 24px; - These are CSS rules written directly in the HTML tag
  • This styling only applies to this one specific element, not to other h1 tags

When to Use This: Only for quick testing. Don't use this for real websites because if you want to change the style later, you have to edit every single element. With external CSS, you change it once and it updates everywhere!

The Cascade in CSS

CSS stands for Cascading Style Sheets. The cascade refers to how CSS rules are applied when there are multiple rules targeting the same element.

When multiple CSS rules apply to the same element, the browser determines which rule takes precedence based on:

  • Importance: Rules marked with !important have highest priority (avoid using this)
  • Specificity: More specific selectors override less specific ones (e.g., #id is more specific than .class)
  • Source Order: Later rules override earlier rules if specificity is equal
/* Example of CSS cascade */ p { color: black; } /* Least specific */ .intro { color: blue; } /* More specific (class selector) */ #main { color: red; } /* Most specific (ID selector) */

Key CSS Concepts

  • Selectors: Patterns that target HTML elements to style
  • Properties: CSS attributes that control styling (color, font-size, margin, etc.)
  • Values: The actual values assigned to properties (blue, 24px, 10px 20px, etc.)
  • Specificity: How CSS determines which rules apply when multiple selectors target the same element
  • Inheritance: Some CSS properties are inherited from parent elements to child elements
  • Box Model: Understanding margin, border, padding, and content area
  • Display Property: Controls how elements are laid out (block, inline, flex, grid)

Browser Support and Compatibility

Most modern browsers (Chrome, Firefox, Safari, Edge) support all the CSS features covered in this course. Older browsers like Internet Explorer 11 have limited support for modern CSS features like Flexbox and Grid. When building professional websites, always test your CSS across multiple browsers.

You can check browser support for any CSS feature at caniuse.com, which shows exactly which browsers support specific CSS properties.

Best Practices for CSS Beginners

  • Use External CSS: Keep CSS in separate files for better organization and reusability
  • Avoid Inline Styles: Inline CSS is hard to maintain and should be avoided
  • Use Meaningful Class Names: Name classes based on what they do, not what they look like (e.g., .button-primary, not .blue-button)
  • Keep CSS Organized: Group related styles together and add comments to explain your code
  • Use Consistent Indentation: Proper formatting makes CSS easier to read and maintain
  • Minimize Specificity: Use simple, low-specificity selectors to make your CSS easier to override and maintain

Common CSS Units

CSS supports various units for sizing elements. Understanding the differences is important for responsive design:

  • px (pixels): Fixed units. 1px = 1 pixel on screen. Good for precise sizing but not responsive.
  • em: Relative to parent element's font size. Useful for responsive typography.
  • rem: Relative to root (html) font size. More predictable than em. Recommended for modern layouts.
  • % (percentage): Relative to parent element. Great for responsive widths and heights.
  • vw/vh: Viewport width/height. 1vw = 1% of viewport width. Useful for full-screen designs.
