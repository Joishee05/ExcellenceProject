HTML Entities

Lesson 11 of 15

HTML entities are special codes for characters that have meaning in HTML or don't appear on standard keyboards. Entities begin with & and end with ;. They let you display symbols, special characters, and reserved characters correctly.

What Are HTML Entities?

HTML entities are text sequences that represent characters. Some characters are "reserved" in HTML (like < and >), while others are difficult to type. Entities solve both problems by providing alternative representations.

Why Use Entities?

  • Display reserved characters (<, >, &, ")
  • Insert special symbols (©, ™, €, etc.)
  • Display non-ASCII characters from other languages
  • Improve document compatibility

Common HTML Entities

&lt; = < (less than)

&gt; = > (greater than)

&amp; = & (ampersand)

&quot; = " (quotation)

&apos; = ' (apostrophe)

&copy; = © (copyright)

&reg; = ® (registered)

&trade; = ™ (trademark)

&euro; = € (euro)

&pound; = £ (pound)

&nbsp; = non-breaking space

&deg; = ° (degree)

Reserved Characters in HTML

These characters have special meaning in HTML and must use entities:

Reserved Character Examples:

<!-- Display a less than symbol: -->
&lt; 5 and &gt; 10
<!-- Displays as: < 5 and > 10 -->

<!-- Display ampersand: -->
Tom &amp; Jerry
<!-- Displays as: Tom & Jerry -->

Numeric Character References

Instead of named entities, you can use numeric references. Two types:

Decimal: &#number;

&#169; = © (copyright symbol)
&#169; = © (same symbol)
&#60; = < (less than)

Hexadecimal: &#xhex;

&#xa9; = © (copyright)
&#x3c; = < (less than)

Special Cases

  • &nbsp; – Non-breaking space (prevents line break)
  • &nbsp;&nbsp;&nbsp; – Create indentation
  • &shy; – Soft hyphen (optional break point)
  • &zwj; – Zero-width joiner

Entity Use Examples

<p>Price: &pound;25 (£25)</p>
<p>Copyright &copy; 2024</p>
<p>Temperature: 98&deg;F</p>
<p>A &lt; B</p>
<p>Kodak&reg; TM</p>

Best Practices with Entities

✓ DO
  • ✓ Use entities for reserved chars
  • ✓ Use UTF-8 encoding
  • ✓ Use named entities when possible
  • ✓ Be consistent with encoding
  • ✓ Use &nbsp; sparingly
✗ DON'T
  • ✗ Forget & in & text
  • ✗ Use entities for regular text
  • ✗ Mix entity styles inconsistently
  • ✗ Use &nbsp; for spacing
  • ✗ Forget closing semicolon

Why HTML Entities Matter

Understanding HTML entities is crucial because: Entities ensure proper display of special characters and reserved symbols. They maintain compatibility across different browsers, languages, and encodings. Proper entity usage prevents HTML parsing errors and ensures content displays correctly for all users worldwide.
Key Takeaways
  • Entities use & (ampersand) and end with ; (semicolon)
  • &lt; displays <, &gt; displays >, &amp; displays &
  • &quot; displays ", &apos; displays '
  • Common symbols: &copy;, &reg;, &trade;, &euro;
  • Numeric entities: &#number; (decimal) or &#xhex; (hex)
  • &nbsp; creates non-breaking space (use for layout)
  • Always include semicolon at end of entity
  • Use UTF-8 encoding for best compatibility
